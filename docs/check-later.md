# 耳確認＝1回flush用リスト（出先後にまとめて聴いて潰す）

**運用**：このリストは「1回のflush（まとめ聴き）」で判定する単一ポイント。上から聴いて OK/要直し を記入→直したら消す。
旧itemは骨格エンジンの大幅進化で陳腐化したため現状に集約（2026-06-28）。生成物は会話送付 MIDI（再生はユーザー側）。

## ★flush本体：現production メロを聴いて判定
現production＝**長音抽出の学習骨格＋動機反復(0.85)＋音域中心保持＋長短別モデル＋句頭休符＋倚音(0.5)**。
（八分の motif-driven 版はまだ scratch prototype＝下の「保留」）。

聴いて一言ずつ：
1. **骨格の質**：構造線がコードに乗る・方向感・長短の差・MIN2の導音。音域暴れが無いか。
2. **動機反復(0.85)**：「同じ動きが戻る」identity が出てるか／**くどくないか**（弱め→下げ/強め→上げ＝repetition param）。
3. **倚音(0.5)**：強拍の表情になってるか／**多すぎ(mannered)でないか**（0.5〜0.9 を耳で選ぶ＝generate.ts `?? 0.5`）。
4. **句頭休符**：遅延入場の「間」が自然か／無音20%が過剰でないか（過剰→`SKELETON_REST_BY_POS`弱める）。
5. **総合**：メロとして成立してるか・濁り（伴奏ボイシング起因か要切り分け）。

## ★flush後に効くパラメータ（耳で値を決める）
| param | 既定 | 耳で決める |
|---|---|---|
| repetition | 0.85 | 反復の強さ（弱→0.6 / 強→1.0） |
| appoggiatura | 0.5 | 強拍倚音率（実曲CT57%なら~0.9） |
| rangeSteps | 10 | 音域（6度に絞る→6） |
| 句頭休符量 | SKELETON_REST_BY_POS | 多すぎ→係数で弱める |
