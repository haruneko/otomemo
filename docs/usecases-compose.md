# チャット作曲ユースケース（生きた文書）

最終更新: 2026-06-24

**この文書は使いながら直す。** 実際に Claude チャット＋creative-manager MCP で作曲してみると「この問いの切り方が微妙」「この動きが抜けてた」が必ず出る。出たらここを更新する（コードや道具設計より上＝使い方が先）。

関連：上位＝`requirements.md` L95-121（頼み方：枠×動作）/ #92,#151（AIは引く・選ぶ・直す／当てずっぽうを出さない）。道具設計＝`design.md #100/#101`。素材＝`research/2026-06-22-association-usecases.md`（連想8機構 M1-M8）/ `research/2026-06-21-routing-scenarios.md`。

## 体験の芯（要件から）
ゼロから音を生む装置ではなく、**自分の素材＋取り込んだ素材から「連想して引き／選び／手直しし／理由を教える」**相棒。AIは「何を・どんな雰囲気で」を汲む役、音楽的妥当性は確実な仕組みが担保（#92）。根拠なきものを勝手にこしらえない（#151）。

## 実ワークフロー（オーナーの言葉・2026-06-24）
> 入口は**それっぽいコード進行＋雑なメロ**があればよい。詰める時に**変形**を使う＝「この進行の**後ろをオープンに**」「メロが合わないから**合わせて**」。最終的に**ネタを並べたから整えて**。
> あと「**YOASOBIパクりたいからAメロ用の進行を4小節2つ繋がるように**」みたいな様式・役割・構造の指定も。

→ 中心の目的動詞：**置く・寄せる・合わせる・整える**（＋**引く**＝連想が中核）。

## 引く系の鉄則：見つけるか、無いと言う（#151）
「切ない進行」「あの曲のコード」「YOASOBIっぽく」は**捏造しない**。①ネタ帳コーパス（アーティスト進行が入っている＝U-FRET取込、例 `Mr.Children-Again` source/tags付き）を引く → ②薄ければ Claude の Web 検索 → ③それでも無ければ「見つかりませんでした」。

## チャットMCPの対象外（別経路）
- **再生（聴く）** ＝ web アプリ（Tone.js/SF2）。
- **DAW(ABILITY)書き出し** ＝ MIDI 書き出しフロー。
- **重い多段の留守番** ＝ 必要なら非同期（が、会話で駆動する分はチャットで完結させる）。

## 未確定・要・実使用検証
- U9「オープン」のような**感じの語彙**は実際に使うと足りない/ズレる可能性大。使って増やす。
- U18「きれいに」の**仕上げレシピ**（境目→調→衝突の順、どこまで自動か）は実演で詰める。
- U5「役割/構造」の解像度（Aメロらしさ＝密度/音域/終止？）は使いながら。
