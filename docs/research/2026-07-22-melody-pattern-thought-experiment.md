# 思考実験：メロディもH2（相対パターン）で作れないか（2026-07-22）

**発端**＝オーナー起票。同日確定の設計原則 H2（正典＝`2026-07-22-performance-editing-architecture-audit.md`）——「伴奏系ネタの正体＝相対パターン（度数×グリッドの演奏の仕方）。文脈（進行・調）に当てて良きように鳴る。編集の家＝ネタエディタでパターンを編集」——に対し、オーナーの直感：**「メロディも同様にできないか。多分できないんだけど」**。この直感を検証する。
**性格**＝Fable による実装なしの設計思考実験。既存 research doc・design・実コードとの突き合わせのみ（新規実測なし）。忖度なし＝「できる」に倒さない。

---

## 0. 結論（先出し）

**オーナーの直感は正しい。メロに H2 は外挿できない。ただし「できない」のは content の層だけで、層分解の下層（リズム・モチーフ図形・骨格テンプレ）には相対パターンが既に効いており、しかも既存機構でほぼ足りている。**

- **(a) 本質的理由**：H2 が成立する4前提（逐語反復がグルーヴ／度数解決しても同一性が保たれる／良さの判定が局所／人が直したい単位＝パターン）を、メロは**4つ全部**破る。前景の同一性は一回性の音列そのものにあり、良さは大域的（句アーチ・カデンツ・フォーム回帰・歌詞）で、反復の文法は**タイルの逆**（隣接の逐語反復≈1%＝近くで変え・遠くで戻す）。パターンを content にすると、破綻するだけでなく手編集主権（仕上げは人間）が resolver の下に潜って消える。
- **(b) 部分的に効く層**：リズムパーツ層（S4）＝**そのまま H2 のパターン**で実装済み・足りている。モチーフ図形（Motif16/seedMotif）＝相対パターンだが敷き方がタイルでなく**変奏文法**＝種として機能・実装済みで足りている。骨格層＝H2 的性質（相対・配置時解決・複数セクション再利用）を最も多く持つが、型（輪郭テンプレ・スキーマ・skelForm）は**content でなく生成バイアス**としてのみ効く——これも実装済み。**「やれる残り」は実質なし**（kind="motif" 一級化だけが#20の予約席として残るが、H2圧力は弱く実需待ちが正当）。
- **(c) 追記案**：§6 の一文。統一原理＝**「content は人が仕上げる単位」**。背景はパターンを仕上げる→パターンが content（H2）。前景はノートを仕上げる→notes が content。前景/背景の区別は恣意でなく、この同一原理の二つの帰結。

---

## 1. 伴奏がパターンで成立する原理（H2 の4前提の言語化）

H2 が「良きように鳴る」のは偶然でなく、伴奏系パートが次の4前提を満たすからだと分解できる。

| # | 前提 | 中身 | 伴奏での姿 |
|---|---|---|---|
| **P1 反復性** | パートの同一性＝「演奏の仕方」であり、**逐語反復が音楽的に正**（グルーヴの条件） | ドラムパターンは小節ごとに verbatim 反復してこそノリ。1〜4小節のタイルで全曲を覆える | rhythm の lanes・chord_pattern の hits がそのまま何周も鳴る |
| **P2 文脈解決の無害性** | 度数→実音の解決で同一性が壊れない。進行が変わって実音が変わっても「同じパターン」と聞こえる | ベースの R-5-R-approach は C でも Am でも「同じ弾き方」 | だから同じパターンを Aメロとサビ（進行違い）で共有できる＝監査§5「共有と相性が良い」 |
| **P3 判定の局所性** | 良さ＝「今のコードとグリッドに合っているか」で決まる。句レベルの目的論（どこへ向かうか）を持たない**定常テクスチャ** | 伴奏は状態であって過程でない。8小節目のコンピングが1小節目より「クライマックス」である必要はない | resolveChordPattern/resolveRelativeBass は小節局所の純関数で完結する |
| **P4 編集単位の一致** | 人が直したい単位＝パターン1周。「1小節直せば全小節に効く」が望みどおり | 「2拍目を抜きたい」は全周に効いてほしい | だから pattern-as-content が手編集主権と**一致**する＝H2 の家はネタエディタ |

先行エンジン（BiaB/Yamaha SFF/MMA＝`2026-07-22-accompaniment-style-engines.md`）が全員この形（固定パターン断片＋コード適応規則）なのは、P1〜P4 が伴奏の物理だから。

## 2. メロはどの前提を破るか——全部

- **×P1（反復の文法が逆向き）**：POP909 全909曲の実測（`2026-07-14-motif-transform-stats.md`）＝**隣接距離での逐語反復シェア 1.4%/0.4%**。直後にもう一度言うときは必ずどこか変える（音数ジッタ±1〜2が6〜7割）。逐語反復が解禁されるのは5-8小節・セクション回帰の**遠距離のみ**（14〜16%）。つまりメロの反復文法は「近くで変え・遠くで戻す」＝**タイルの正反対**。パターンを rotate で敷いた瞬間、ポップのメロが最も避ける「隣接逐語反復」を量産する。伴奏では同じ操作が正になる——ここが前景/背景の実測上の分水嶺。
- **×P2（前景の同一性は音列そのもの）**：聴き手はメロを音符単位で追跡する（figure/ground の figure）。同じ相対パターンを別進行に当てた2つの実音化は、伴奏なら「同じパターン」、メロなら「別のメロ or 変奏」と聞こえる。リズム保存・音高再フィット（rhythm_keep_pitch_var）は実測で**変換の8.4%**＝変奏の一級技法ではあるが「全体の作り方」ではない。度数解決に耐えるのはリズム＋輪郭の図形層だけ（→§3(b)(c)）。
- **×P3（良さは大域的）**：メロの良さは句アーチ（凸が最頻・頂点位置0.55-0.66＝`2026-07-14-contour-template-dictionary.md`）・カデンツ着地・クライマックスの一回性・フォーム回帰（A A' A）・歌詞プロソディで決まる＝**軌道（trajectory）であって状態でない**。実装がこれを証言している：genMotifMelodyV2 の文脈入力は chords/key に留まらず drums（drumLock/backbeat）・bass（counter）・骨格 phrases・section role・prevEndPitch・lyrics（WP-L0）まで広がり、14段後処理が句・終止・表情を大域に整える。伴奏の resolver（小節局所の純関数）とは複雑性のクラスが違う。「パターン×局所解決」に畳める構造ではない。
- **×P4（人が直したい単位＝ノート）**：メロは人が仕上げるパート。「3小節目のこの音だけ上げたい」が編集の基本単位で、「直せば全小節に波及」は**望まない**。メロを pattern+resolver の content にすると、手編集は (i) resolver の出力に対して行えば再解決で消える (ii) パターンに対して行えば全反復に波及して一回性が作れない——どちらも「仕上げは人間」と衝突する。監査§1 の作曲層の行（「パラメータを正準にすると手編集が再生成で消える」→メロは焼き込みが正当）はこの帰結を既に言っていた。本思考実験はその**理由**を4前提の破れとして明晰化した。

## 3. 層分解との突き合わせ——層ごとの判定

メロは単層ではない。Otomemo の層分解に H2 の「パターン」概念を当てると、判定は層ごとに割れる。

| 層 | H2 の「相対パターン」に相当するか | 判定 | 根拠 |
|---|---|---|---|
| **(表面) content=notes** | ✗ 一回性の音列＝作曲層 | **パターン化不可**（§2 の4つの破れ全部がここに集中） | 監査§6「メロ・リフ・counter へ広げた瞬間に破綻」の再確認＋理由の明晰化 |
| **(b) リズムパーツ層（S4 rhythmParts）** | ◎ **文字通り H2 のパターン**：x/. 16枠グリッド＝ドラム RHYTHM16 と同一表現・rotate＝「ドラムパターンを Section に敷く感覚」（design S4-1 の自己記述）・音高は骨格/コード機構が解決 | **パターン化できる＝実装済み・足りている** | `rhythmParts.ts`（プリセット10・L1 rotate・L2 placement・extractRhythmPart 採取・custom）。メロで唯一タイルが成立する層＝**音高を運ばない**から P2 を破らない。逐語反復も、リズムだけなら伴奏側の文法（グルーヴ）に属して正 |
| **(c) モチーフ図形（Motif16/seedMotif/keepFirstBlocks）** | ○ 相対パターンそのもの＝「相対onset＋度数move・実音を持たない→移高で和声適応」（design #20）。H2 の定義文とほぼ同文 | **条件付き：パターンだが、敷き方がタイルでない**。種（seed）として機能し、展開は変奏スケジューラ（skelForm WP-M2＝距離条件付き literal 率・cadence-swap・A A' A 原型回帰）を通る。content にはならない | `melodyCells.ts` extractMotif16:512/seedMotif:946〜。実装済みで足りている |
| **(a) 骨格層（#20 skeleton neta）** | △ H2 的性質を最も多く持つ：独立素材ネタ・配置時オンザフライ移調・**同じ骨格を複数セクションへ置き表面化だけ変える再利用**（chord_progression と同じ建付け）＝「文脈に当てて鳴る」の骨格版 | **条件付き：再利用単位ではあるがタイルしない一回性の軌道**。content はブレークポイント列のまま。型が効くのは生成バイアスの層（下の行） | design #20 決定事項。骨格には編集の家（骨格の机 S6）も既にある＝H1 は骨格については充足済み |
| **(d) 骨格テンプレ辞書（輪郭テンプレ・ガラントスキーマ・skelForm）** | ○ 辞書の形は伴奏の型辞書と同族（型ID×パラメタ×文脈×頻度prior）。ガラントスキーマは「上声度数×バス度数×拍×機能」＝2声の相対パターンそのもの | **型は content に複写せず、引力（λソフト制約）・プロトタイプ・複写計画としてのみ効く**＝実装済み（contour λ=1.0 中間スロットのみ・skelForm 複写＋カデンツ差替） | contour doc §1.4「型は連続空間のプロトタイプ・離散カテゴリ扱い禁物」／galant doc §7-6「**表面はスキーマに任せない**。同一骨格×複数seedで別物にするのが18世紀くささ回避の要」 |

**構図の対比（この思考実験の中心図式）**：
- 伴奏：**型辞書 → 採用 → content＝パターン → レンダ時解決＝そのまま鳴る**。型が完成品の形をしている。
- メロ：**型辞書 → 生成バイアス/種 → 変奏文法 → 表面化 → content＝notes（一回性）→ 人が仕上げる**。型は胚であって、2段の追加変換（変奏スケジューラ＋表面化）を通らないと曲にならない。

つまり「型辞書」は両者に存在する（メロにも既に3系統ある）。違うのは**型の住所**：伴奏では content、メロでは生成の上流。H2 の外挿が失敗するのは「メロに型がない」からではなく、「型を content に置けない」から。

## 4. 実証との整合——motif-transform-stats は H2 外挿を支持するか

**半分支持し、半分反証する。そしてその割れ方が§3の層判定と正確に一致する。**

- **支持する部分（下層のパターン化）**：リズム保存・音高差し替え（8.4%）が音高保存・リズム変形（0.9%）の**約10倍**＝「ポップではモチーフの同一性を**リズムが担う**。音高はコードに合わせて差し替える」。これは H2 の核（度数×グリッドの型を文脈に当てる）がメロの**図形層では実際に起きている**ことの実測。rhythmParts（リズム＝パターン）と Motif16（図形＝相対パターン→refitIdxToChord で和声再フィット）は、まさにこの統計の実装であり、外挿はここまでは正しい。
- **反証する部分（content のパターン化）**：①隣接逐語反復≈1%＝タイル禁止（§2）。②変奏の主要通貨＝音数ジッタ±1〜2（6〜7割）＝表面は常に微変し続ける＝verbatim なパターンが content であり続けられない。③ A A' A 原型回帰（族の8〜9割）＝展開は「距離を見て変え、戻す」**スケジューラ**を要求する＝resolver（無状態・局所）では表現できない。つまり実測は「メロの型は種と文法であって、敷物ではない」と言っている。

## 5. コンセプト整合——「候補まで機械・仕上げは人間」との関係

メロをパターンで敷くことは思想違反か？ **違反である。ただし理由が重要**で、「機械が作りすぎるから」ではない（伴奏では機械のパターンが content になっても違反でない）。違反になるのは**仕上げの単位を間違えるから**：

- 背景パートで人が仕上げるもの＝**パターン**（2拍目を抜く・voicing を変える）。だから pattern-as-content が手編集主権を**実現**する（H2 の家＝ネタエディタ）。
- 前景で人が仕上げるもの＝**ノート**（この音を上げる・ここで切る）。だから notes-as-content が手編集主権を実現する（監査の作曲層＝焼き込み正当）。

つまり H2（伴奏）とメロの焼き込みは、対立する2原則ではなく、**同一原理「content＝人が仕上げる単位」の二つの帰結**。前景/背景の区別は原則に昇格するが、その根はさらに一段深いこの統一原理にある。図地（figure/ground）の知覚区別 → 仕上げの単位の違い → content の形の違い、と因果が通る。

## 6. 結論

**(a) メロに H2 を外挿できない本質的理由（言い切り）**：
H2 は「パートの同一性が反復可能な演奏の仕方にあり、度数解決が同一性を壊さず、良さが局所で決まり、人が直したい単位がパターンである」ことに立脚するが、メロ（前景）はこの4前提を全部破る——同一性は一回性の音列そのものにあり（聴き手が音符単位で追跡する figure）、良さは句アーチ・カデンツ・フォーム回帰・歌詞という大域の軌道で決まり、反復の文法はタイルの逆（隣接逐語反復≈1%・近くで変え遠くで戻す）であり、人が仕上げる単位はノートである。ゆえにパターンを content に据えると、音楽的に破綻する（機械臭い逐語タイル）以前に、設計的に破綻する（手編集が resolver の下に潜って消える＝「仕上げは人間」の喪失）。

**(b) 部分的にパターンが効く層と「やれる残り」**：
効く層は3つ、**すべて実装済みで、既存機構で足りている**。
1. **リズムパーツ層（S4）**＝完全な H2 型パターン（x/.16枠・rotate/placement/採取/custom）。音高を運ばないから唯一タイルが成立する。残＝L2 小節ペイント UI のみで、これは既に backlog 管理下（design S4-2）＝新規の残なし。
2. **モチーフ図形（Motif16）**＝相対パターン（種）＋変奏文法（skelForm WP-M2）＝実測文法どおりの分業が完成している。唯一の未実装は #20 が予約した **kind="motif" の一級ネタ化**（現状 Motif16 は内部転送形式・section.seedMotif 経由のみ・エディタなし）。ただし H2/H1 の圧力で測ると弱い：モチーフは content でなく種であり、効果は生成されたメロの notes に読める形で残る（監査の基準「効果が content に読めるならノブ自体は消えてよい」）＝家の常設は必須でない。**実需（Chat でモチーフを採って回す運用が実際に増える等）が出るまで着手不要**と言い切る。
3. **骨格テンプレ辞書（輪郭・スキーマ・skelForm）**＝型は引力・プロトタイプ・複写計画として効かせる形が既に正しく、編集の家（骨格の机）も既にある。残なし（スキーマ辞書の本実装は galant doc の判定どおり GO-限定つきの将来スライスであって、本思考実験からの新規要求ではない）。

**(c) 設計原則への追記案**（design.md「パターン選択の家＝ネタエディタ」節 or 監査§1 原則文への1項として）：

> **H2 の適用境界（前景/背景の原則）**：H2 は背景パート（進行に当てて鳴る反復パターンが content である kind＝chord_pattern / section_inst / rhythm / bass）の原則であり、前景（メロ・リフ・counter）には外挿しない——前景の content は常に一回性の notes（作曲層・手編集主権）。ただし前景でも層分解の下層には型が効く：リズムパーツ（x/.16枠）・モチーフ図形（Motif16）・骨格テンプレ（輪郭/スキーマ/skelForm）は相対パターンだが、その住所は content でなく**生成の種/バイアス**であり、敷き方はタイルでなく変奏文法（近くで変え・遠くで戻す・A A' A）を通る。統一原理＝**content は人が仕上げる単位**：背景はパターンを仕上げる→パターンが content、前景はノートを仕上げる→notes が content。
