# 16分走句(runs)の同音潰れ修正＝スケール1段移動（2026-07-10）

> 状態＝計画。対象＝`apps/api/src/music/melodyCells.ts` の `render` 走句分岐。4/4・6/8共通。6/8は[[2026-07-10-68-grid-unify-16th]]の統一12枠グリッド上で効く。

## 問題

走句(runs)の16分は**音価(dur)はレガートで繋がる**が、**音程が同音反復に潰れやすい**（実測：連続16分ステップの同音率 4/4=47%・6/8=42%、`64 64 64 64` のような刻み）。スケールを駆け上がる走句になりきらない。

## 原因（コード実体）

`render` 弱拍/走句分岐（:770-775付近）：
- `want = prev + M.mv[i]`（`M.mv` は**半音**オフセット。走句では `rdir`=±1固定）。
- `p = snapList(want, L)`（`L`=スケール音の実MIDIピッチ列）。
- `snapList`（:741）はタイブレーク `d < bd`（厳密比較）＝**同距離なら昇順先着＝低い方が勝つ**。

C major・prev=C(60)・上行(+1)のとき want=61。候補C(60)とD(62)は距離同じ「1」→先着でCに戻る＝**同音潰れ**。フォールバック `if (p===prev) p = snapList(prev±1, L)` も半音再計算で同じ結果＝直らずデッドコード。下行はタイが常に1段下に解決するため潰れない＝**上行走句の全音境界（長調で7段中5段）に潰れが集中**。生成側(mkMotif rdir=±1)は無罪、renderの解釈だけの問題。

## 決定＝案B（走句はスケール1段移動・runsゲート）

走句音を「prevのスケールindexから `sign(M.mv[i])` 方向へ1段」で決める（`nearestIdx`/`clampScale`＝ファイル内に実績あり :885,:1063-1066）。案A（同ロジックを直書き）と走句ピッチは完全同値だが、案Bはフォールバック1行の置換で済む最小パッチ。

**必須ゲート**：4/4は`runs`未指定でも既定語彙に隣接16分が含まれ走句分岐を通るため、無条件書き換えは「runs未指定bit一致」を壊す。**`rns !== undefined && M.run[i]` のときだけ**スケール段移動に切替、それ以外は現行snap維持。6/8既定は走句が無いので元から不変。

- 非走句・強拍CT(`ctOf`)・弧(anchor)・終止・カデンツ・表情には触れない。禁則(|iv|≥5判定)/gap-fillはstep(1-2半音)に反応しないので不変。音域は`clampScale`で保証。決定的（RNG不使用）。
- score(半音積算)との軽微な不整合は決定的でランキングに閉じ実害なし＝改修不要。

## 同音16分の扱い

走句(run=true)は定義上「順次スカラーpassage」＝潰れは完全排除が正。別テクスチャの「意図的な同音16分刻み」は非走句経路（`moveTrans` Markovがmove=0を取る・`foreground`ノブ）で従来どおり温存＝`foreground`と非競合。初手は完全排除を推奨（意味が明快・テストが単純）。

## 手順・受入条件

1. 受入テスト先行（同音率メトリクス）で現状red確認。
2. render走句分岐を`rns!==undefined && M.run[i]`ゲート付きスケール段移動へ（案B）。生成側/score/runDampは無改修。
3. bit一致(⑳/⑨)・禁則/音域/決定性(㉓/⑪)を緑のまま、新メトリクス緑を確認。

**受入条件**：runs=1・seed1-40で走句ペアの同音率が0.47/0.42→**<0.15**（4/4・6/8とも）／走句隣接ペアの音程が1-2半音(step)の率>0.8（跳躍化しない）／onset隣接率㉑/⑩の>0.5維持／禁則ゼロ・音域[58,83]・決定性／**runs/push未指定bit一致（⑳/⑨）維持**。
