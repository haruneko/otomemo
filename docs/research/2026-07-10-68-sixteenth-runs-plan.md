# 6/8で16分が出ない問題＝グリッド拡張の修正方針（2026-07-10）

> **⚠️ 改訂済（2026-07-10）**：本docの「runs>0でのみ12枠へ切替」方式は `c704441` で実装したが、オーナー指摘「6拍子の基底は16分のはず・8分基底が変」により **[[2026-07-10-68-grid-unify-16th]]（16分12枠を常時基底へ統一）に上書き**された。本docは経緯として保存。以下は初手方針の記録。
>
> **状態＝実装済（c704441）だが後継docで置換予定**。

## 1. 現状と原因（4/4とは犯人が違う）

- **4/4**：語彙（`motif16.rhythm16`＝16枠）には16分が**ある**のに選別スコアが抑圧 → Step4で `runs` ノブ（語彙再重み付け＋ペナルティ減衰）により解決済み。
- **6/8（compound）**：グリッドが**1小節=8分6枠**（`RHYTHM68_DATA`＝設計6文字語・`melodyCells.ts:543-550`）で、
  **16分の置き場（.25/.75オフセット）が格子上に存在しない**。さらに：
  - run旗は compound で**全false固定**（`melodyCells.ts:612-613`）＝走句の方向保持ロジックに乗れない
  - `runs` ノブは 4/4 の `rhythmVocab` のみ再重み付け＝**6/8では完全no-op**
  - 表面ノブ swing/push/drumLock/backbeat/converse も全て `!compound` ゲート
- つまり6/8は「抑圧」でなく**「語彙・グリッドのレベルで不在」**。選別緩和では直らず、格子と語彙を拡張するしかない。

## 2. 音楽的ターゲット（6/8の16分はどこに出るか）

1. **ペア割り**＝8分1個→16分2個（経過・刺繍・同音アンティシペーション）。6/8ポップ/ジグで最頻。
2. **付点8分＋16分**（シチリアーナ/跳ねの強調）＝ロングショート。
3. **16分走句**＝拍（付点四分）を16分で埋める走り（6連相当）。稀・runs高値でのみ。

## 3. 方針＝12枠グリッド拡張（`runs`>0 ゲート・既定は6枠のまま）

**本命1本**。表面パス（後付けで8分を割る加飾）も検討したが不採用：
4/4の `runs` は語彙→選別→輪郭の**モチーフ層**で効いており、同じノブが拍子で層が変わるのは設計として歪む。
モチーフ層に入れてこそ反復・断片化・run方向保持と噛んで「動機の一部としての16分」になる。

### 効き（design #12-M へ追記する内容）

- **グリッド**：`runs`（または `density` 高値）指定時のみ、compound の `mkMotif` を**1小節=16分12枠**へ切替
  （`t = bar*3 + s*0.25`）。未指定＝従来6枠 pick68 verbatim＝**bit一致**。
- **語彙**：`RHYTHM68X_DATA`（新設・設計重み）＝既存6枠語を12枠へ機械展開（`x`→`x.`）した基底＋
  16分含有パターン（上記ターゲット1-3を数個ずつ・計15語程度）。`runW`（隣接16分ペア数の再重み付け）と
  `densW` は4/4と同型で積合成。
- **run旗**：compound の全false固定を撤去し、4/4と同じ「隣接≤0.26」判定へ（`runs`未指定時は12枠経路に
  入らないので従来と同値）。→ 既存の run→方向保持・gap-fill にそのまま乗る。**新ピッチ論理なし**（Step4と同じ原則）。
- **選別**：`score()` の `n16`計上を compound 対応（8分格子外＝.25/.75オフセットのonsetを数える。
  現式 `(t*4)%2` は6/8の従来onsetでは常に0なので既定パスは不変）。受入音数帯 `lo68/hi68` に
  4/4の `hiN` と同型の runs 拡張を追加。`runDamp` はそのまま流用。
- **レンダ**：`onMain`（付点四分頭 0/1.5）のCTスナップは不変。ジグ跳ね dur パス（`:756-762`）は
  `Math.min(g, …)` で gap=0.25 に自然対応するが、**16分ペアが0.55跳ねで潰れないか**をテストで確認
  （必要なら run 音は素直に gap 詰めへ分岐）。
- **評価器**：`corpusTypicality` は6/8でリズム項0（判定不能ガード済・`evalMelody.ts:103-105`）＝クラッシュ/劣化なし。E-rule 側は既存propertyテストで確認。
- **配線**：`generate.ts` 透過・`mcp.ts` の gen_melody 説明文から「6/8はruns対象外」の含意を除去
  （ドラム3ノブの6/8除外は据え置き）。

### やらないこと（スコープ外）

- swing の6/8対応（ジグ跳ねが既にあるため対象外のまま）・push/drumLock/backbeat/converse の6/8対応（別件・backlog）。
- コーパスからの12枠統計抽出（irish等に6/8実データあり）＝**後続の任意フェーズ**。まず設計語彙＋耳で較正し、
  物足りなければデータ差替（4/4 Step4 Phase3 のPOP909再計測と同じ扱い）。

## 4. 受入条件（テスト先行・赤→緑）

1. `runs` 未指定・6/8 → 既存出力と**bit一致**（既定パス不変）。
2. `runs=1`・6/8・seed sweep → 16分onset率（8分格子外の割合）> 0.1、かつ走句性（隣接≤0.26ペア存在）。
3. 決定性＝同一seed同一出力。既存property（音域・禁則ゼロ・単一頂点・dur>0・重なりなし）が runs>0 でも緑。
4. ジグ跳ね dur と16分ペアの共存＝ペアの前音 dur ≤ gap（音が次を食わない）。
5. `corpusTypicality` が6/8 runs>0 でクラッシュせず score劣化なし（moveのみ測る前提の確認）。
