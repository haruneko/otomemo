# ボカロ文法の仕様化 — ボカロ曲の「作られ方」の統計的特徴

- 作成: 2026-07-14
- タスク: G1（ボカロ文法＝人力歌ものとの統計差の仕様化）
- スコープ: 曲全体の「作られ方の文法」。**人声の歌唱制約（音域/tessitura/ブレス）とボカロでの緩和（C6開放・170-185BPM 等）は既知・別調査済**なので本書では前提として扱い深追いしない。
- 手法: WebSearch/WebFetch＋研究者知識。数値は出典に紐づく（末尾に一覧）。分析記事・計量分析・学会論文を優先。

---

## 0. 要旨（設計者向け3行）

1. ボカロ文法は「**人間の喉の制約を外した書法**」の総体。速い（BPM高め）・詰める（音節密度高・16分早口）・跳ぶ（大跳躍/同音連打）・型破り（極短イントロ/頭サビ/非4小節反復/多用転調）が相場。
2. ただし「**全部が全部速い**」は誤解。大規模データの中央値は 132〜145 BPM で、人力J-POPよりやや速い程度。**尖った高速曲（170-200+）と、じっくり系が二極**でシーンを作る。
3. ツール実装は「**ボカロモード**＝既定値をこの相場へ寄せたプリセット」でよいが、**尖り値（詰め込み/跳躍/転調）はノブで振れる**ことが本質。既定を固定した瞬間に文法の魅力（逸脱の自由）が死ぬ。

## 1. テンポ／尺（BPM分布・年代推移・短尺化）

### 1.1 BPM の相場と「高速化」の実像

| 集団 | BPM中央値 | 出典 |
|---|---|---|
| 昭和歌謡 | 124.7 | Tsurezure Lab（Spotify）|
| ボカロ曲（比較データ） | 145.0（昭和比 +20） | 同上 |
| ボカロ曲（大規模: 2007–2022, 約10,054曲） | **132** | Tsurezure Lab 別分析 |
| ハイテンポ系ボカロ（制作Tips実務値） | **170–185**（最多）／範囲160–200 | コアミュージックスクール |
| ボカロック（尖り上限） | **200超も** | Soundmain / RAG Music |

**重要な含意（誤解の是正）**：大規模コーパスの中央値は 132〜145 BPM で、俗説の「ボカロ＝爆速」ほど平均は高くない。しかも 2007–2022 の時系列では「**とくに加速している様子は見られなかった**」（Tsurezure）。一方でアニソン/ボカロ圏の言説では「89秒のTVサイズに展開を詰めるためBPMが上がった」という**尺起因の高速化**が語られる（Real Sound）。
→ **相場の中心（120-145）と、高速サブジャンル（170-200+）を分けて扱う**のが正しい。ボカロ性を最も強く示すのは「中央値の高さ」より「**高速テールの厚さ＋詰め込み**」。

### 1.2 尺と短尺化・詞の増量

- 2007–2022 の時系列で「曲は**徐々に短くなる**」一方、「**歌詞の言葉数は増える**」（Tsurezure、P<0.0001 で有意）。＝**同じ尺により多くの音節を詰める**方向の圧力。これが後述の音節密度上昇と表裏。
- TikTok/YouTube Shorts との親和性が高く、**頭サビ・キャッチ即出し**の設計が短尺文化と噛み合う（kadenzp 等）。TikTok が60秒→3分に伸び「短めの曲はフルで入る／長い曲は1番だけ」運用。
→ **含意**: サビの位置を前に寄せ、"掴み"が15–30秒以内に来る構成が有利。曲尺プリセットは J-POP標準(4–5分)よりやや短め寄りを既定に。

## 2. 構成（型破りな形式）

観測される頻出型（分析記事＋作曲講座ベース）:

- **極短イントロ / 頭サビ（サビ頭・サビ出）**: イントロ/アウトロを削り、歌い出し・盛り上がりを前景化。冒頭からキャッチーで強インパクト。
- **間奏＝ボカロの休み＝インストの聞かせ場**: 歌がないブロックでギター/シンセのソロやリフを聴かせる（ボカロック/ピアノロックで顕著）。ボーカルが機械なので「歌の休符」を器楽の見せ場に転用しやすい。
- **落ちサビ**: ラスサビ直前にテンションを一度落とす静かなサビ。最後の爆発の助走。
- **ラスサビ転調**（次節、数値あり）。
- **型破り要素**: 頻繁な転調、**4の倍数でないフレーズ反復**（人間の呼吸単位に縛られない）。「従来ポップスより型破りな形式が許される」（musicmusicologic / core-ms）。

→ **含意**: セクションプリセットに「intro=極短 or なし」「頭サビ」「間奏=器楽ソロ枠」「落ちサビ」「ラスサビ転調」をフラグとして持たせる。フレーズ長を4小節固定にしない（3・5・6・7小節反復を許容）。

## 3. メロ統計（詰め込み・音域・跳躍・同音連打）

### 3.1 計量分析の土台（学会論文）

- **「ボーカロイドの人気曲における歌詞とメロディの関係の解析」**（情報処理学会, 2013）: ニコニコ週間ボカロランキング（2007–2011）を対象に、**音節あたりの音価・発音単位の長さ・隣接音の音高差・音域・テンポ**を解析。結果、**上位曲はテンポ・"詞乗せ速度"（発話速度）・リズムの変化量に顕著な特徴**を持つ、と報告。＝人気ボカロ曲は「速く・詰めて・リズムを動かす」方向に偏る、という計量的裏付け。

### 3.2 詰め込み（音節密度）

- 170–185 BPM ×16分音符グリッドで「1拍に16分が4つ」入り、ボーカルラインに**詰め込み感＝早口フレーズ**が生まれる（core-ms）。8分では詰め込み感が出にくく、**16分主体が書法の核**。
- wowaka（2009–）に代表される「**BPM159・16分早口歌唱**」が高速詞乗せの原型として言及される。
- 前述の「尺短縮×詞増量」（§1.2）と同じ圧力。**1音節=1音（シラビック/speech-like）**を高速で敷き詰めるのがボカロの詞乗せ。

### 3.3 音域幅・跳躍

- 女声ボカロの実務レンジ: **C4–G5 を基本、サビで A5/B5** へ（core-ms）。人力の緩和という既知事項の上に立つ。
- **大跳躍を意図的に挿入**: 長9度・短10度・オクターブ跳躍を「**人間離れした速度感**」の演出として書く（core-ms / musicmusicologic）。人力ならブレイク＝別テイクになる音程移動を1フレーズ内で連続させる。

### 3.4 同音連打（機械発声を活かす書法）

- ボカロは調声しても**発音に機械的な節回しが残る**。これを欠点でなく個性として使う（musicmusicologic / music-hearts）。**同音連打（ピッチ据え置きで音節だけ刻む）**は人声だと単調に聞こえがちだが、機械発声＋高速では逆に**打楽器的・畳みかけ**として機能する。
- 一般ポップスの通時トレンドとも符合: 米チャート1950–2023の計量研究は「**旋律が年々よりリピティティブ／speech-like（音域が狭く・同音反復が増える）**」方向を報告（Nature Sci. Rep. 2024, Hamilton & Pearce 系 BiMMuDa）。ボカロは**この speech-like 化を先鋭化・機械で最適化した形**と位置づけられる。

→ **含意（メロ生成の既定差分）**: ボカロモードでは (a) 音節密度の上限を上げ16分を許容、(b) 跳躍幅の許容を広げ大跳躍にペナルティを掛けすぎない、(c) 同音連打の連続許容数を増やす、(d) 音域上限を人力より上（C6等の既知緩和）へ。**人力モードはこれらを喉基準で締める**。

## 4. サブジャンル文法（型）

| サブジャンル | 型（サウンド/リズム/メロ） | 代表系譜 |
|---|---|---|
| **ボカロック（VOCAROCK）** | 速いBPM（時に200超）・鋭角ギター/シンセリフ・4つ打ち or 裏打ちハイハットのダンスロック的ビート・急な跳躍・16分早口。ボカロ性を最も凝縮 | wowaka 系〜現行 |
| **ミクノポップ / VOCALOEDM** | 電子音主体・四つ打ち・キラキラ系シンセ。kz(livetune)→…→ツミキ「フォニイ」の系譜（Real Sound） | kz, ツミキ 等 |
| **カワイイ系エレクトロ / エレクトロスウィング** | 明るい電子音・スウィングリズム＋ピアノバッキング、リードに金管（sax/tp）を使う型もあり | — |
| **ピアノロック** | 感情表現をピアノ主導で。間奏のピアノ/器楽ソロが聞かせ場 | — |

- 共通の下地: 「**ボカロ音声を主役に立てる編曲**」（主役パート交代・オケ音量抑制・ハモ追加）。全セクションでボカロを前に出すミックスが前提（musicmusicologic）。

→ **含意**: サブジャンルは「BPM帯 × ビート型（4つ打ち/裏打ち/バンド8ビート）× リード音色（歪ギター/シンセ/ピアノ/金管）× メロ尖り度」の直交パラメータで表現できる。ボカロックは全パラメータを尖り側へ、カワイイ系は電子＋明るい調へ、というプリセット差で足りる。

## 5. 歌詞テーマ／語彙（軽く）

- 本題外のため要点のみ: 内面・葛藤・厭世/自己言及・言葉遊び（韻・母音配置）が多いと語られるが、**計量的に強い一次データは薄い**。確実に効くのは「**語数の増量＝詰め込み**」という形式側の事実（§1.2）で、テーマ語彙は生成の既定値には落とし込まない（テンプレ化の弊害大）。

## 6. 調性／和声（頻出進行の相場）

- **王道進行 4536**（IV→V→iii→vi）: J-POP/アニソン/ボカロの三大定番の一角。ボカロでも多い。
- **小室進行 6451**（vi→IV→V→I, Cなら Am→F→G→C）: 短調始まりからメジャーへ開くドラマ性。ボカロで頻出。
- **丸サ進行**（IVM7→III7→VIm7→Vm7→I7, 非ダイアトニック含む）: おしゃれ/都会系で使用。
- **借用和音**（♭VII 等）や平行短調への移行を織り込む型（core-ms 実例①②③）。
- **ボカロ伝説入り曲の傾向**（ニコニコ大百科の分類）: **王道進行が多い＋小室進行はじめ短調曲が非常に多い**。逆に**カノン進行などメジャートニック始まりは比較的少ない**。＝人力J-POP比で**短調・マイナー基調へ寄る**のがボカロ和声の相場。

### 6.1 転調の相場（数値）

若葉の分析（1,101曲, 2021-09 時点、平行調転調は除外）:

| 転調タイプ | 曲数 | 割合(1101中) |
|---|---|---|
| **ラスサビ転調** | 439 | **約39.9%** |
| サビ入り時転調 | 284 | 約25.8% |
| 落ちサビ転調 | 62 | 約5.6% |
| サビ途中転調 | 17 | 約1.5% |
| 1番/2番で異調 | 5 | 約0.5% |

- **約4割がラスサビで転調**は極めて高い。J-POP全般でも「ラスサビ半音上げ」は世界比で異常に多用（aura-voice / SoundQuest）だが、ボカロはさらに転調全般を型として抱える。定番は**サビ手前で半音上げ**て高揚を作る（core-ms）。

→ **含意**: ボカロモードは (a) 短調/マイナー基調をやや優先、(b) 6451・4536・丸サをコード提案の上位に、(c) **ラスサビ転調（+1半音 or 全音）を既定オプションでON**（人力モードはOFF既定）。

## 7. 仕様化：「ボカロモード」プリセット（人力歌ものモードとの差分表）

> 既定値＝相場の中心。**尖り値はノブで振れる**ことが本質（§0-3）。表は「どこをどれだけ緩める/締めるか」の差分。

| 軸 | 人力歌ものモード（既定） | **ボカロモード（既定）** | 根拠 |
|---|---|---|---|
| BPM 既定帯 | 90–130 | **130–175**（高速サブは170–200+） | Tsurezure(132/145) / core-ms(170-185) |
| BPM 上限（許容） | 〜160 | **〜210** | ボカロック200超 |
| 音価グリッド | 8分主体 | **16分主体（詰め込み許容）** | core-ms |
| 音節密度（1拍あたり音節上限） | 低〜中 | **高（16分早口を許容）** | IPSJ2013 / core-ms |
| 音域（女声既定） | 喉基準（〜G5目安） | **C4–B5、上限C6まで開放（既知緩和）** | core-ms（既知前提） |
| 跳躍許容 | 跳躍にペナルティ強（ブレス/歌唱性） | **長9度・短10度・8度連続を許容** | core-ms / musicmusicologic |
| 同音連打 | 連続を抑制（単調回避） | **連打を積極許容（機械発声を活用）** | musicmusicologic / Nature2024 |
| フレーズ長 | 4小節基調 | **非4小節反復を許容（3/5/6/7）** | musicmusicologic / core-ms |
| イントロ尺 | 標準（4–8小節） | **極短 or なし／頭サビ許容** | 構成分析 |
| 間奏 | 器楽 or 歌 | **器楽ソロ枠（歌の休符を聞かせ場化）** | サブジャンル文法 |
| 落ちサビ | 任意 | **ラスサビ前に既定で候補提示** | 構成分析 |
| 曲尺 | 4–5分 | **やや短め寄り／掴み前倒し** | Tsurezure短尺化 / 短尺文化 |
| 調性の傾き | 長調/短調ニュートラル | **短調・マイナー基調へやや傾ける** | ニコニコ大百科(伝説入り分類) |
| コード提案の上位 | カノン/王道 | **6451・4536・丸サ・♭VII借用** | 各進行解説 / core-ms |
| ラスサビ転調 | 既定OFF | **既定ON（+1半音 or 全音）** | 若葉(約40%) / core-ms |

### 7.1 実装メモ（このツールへの落とし込み）

- **モードは「差分プロファイル」として持つ**: 既存の生成器（gen_melody / gen_chords / assemble / セクション役割）に、上表の各軸を係数/上限/フラグで注入する形。新エンジンを作らず、**重み・許容域・既定フラグの差し替え**で表現できる（＝直交パラメータ、§4）。
- **「既定を固定しない」**: ボカロ文法の魅力は逸脱の自由。プリセットは出発点で、密度/跳躍/転調/フレーズ長のノブを前面に出す（本ツールの設計思想「機械は候補・仕上げは人間／選択肢とばらつきに振る」と整合）。
- **中央値と尖りの二層**: BPM/密度は「相場の中心プリセット」と「高速・高密度プリセット（ボカロック）」の2枚を用意し、単一既定にしない（§1.1 の誤解是正）。
- **和声は短調寄り＋定番進行を上位に**、ただし提案止まり。転調は「サビ手前/ラスサビ」の位置フラグとして候補提示。

## 8. 未解決・要追調査（正直な留保）

- **音節密度・跳躍・同音連打の"人力比"を数値で**確定できていない（IPSJ2013 は特徴の存在を示すが公開本文の実数まで未取得。CiNii/情報学広場に本文あり＝要精読）。プリセットの具体閾値は暫定。
- Tsurezure の「加速していない」と実務言説の「高速化」は**データセット/母集団の違い**（Spotify流通分 vs ニコニコ全体、対象年）。年代別の厳密な BPM 推移は一次データで未確定。
- ラスサビ転調 約40% は単一分析者(若葉)の集計。J-POP対照群との統制比較ではない。
- speech-like 化（Nature2024/BiMMuDa）は**ポップス全般の通時トレンド**で、ボカロ固有性の切り分けは今後の課題。
