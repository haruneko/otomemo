# メロディ生成 研究仕様（フレーズ感・息継ぎ／連想記憶への道）

最終更新: 2026-06-23 ／ 種別: **研究→仕様（実装前・レビュー用）**。3視点パネル（理論→決定的ルール／計算手法サーベイ／メロ連想・エッセンス）を統合。
位置づけ: これは「非常に大きな機能（メロの連想記憶）」の**1個目**。北極星＝**参照メロ（例：椎名林檎）のエッセンスを引いて"違うメロ"を生成**。

## 0. 現状診断（なぜ「呼吸しない」のか）
`apps/api/src/music/generate.ts` の `genMelody` は「1小節モチーフを全小節に反復＋拍頭コードトーン拘束＋軽い変奏」。良い土台だが、**phrase/period(2小節/4小節)の上位構造・句末の休符/長音・終止の安定音着地・頂点(クライマックス)設計がいずれも構造的に欠落**＝延々続いて息継ぎしない。原因はモチーフ展開でなく**上位の"呼吸の骨格"が無い**こと。

## 1. 確定した設計原則（3視点が収束）
1. **骨格優先（skeleton-first）**：音を作る前に**フレーズ構造＝呼吸を先に置く**。`motive(1小節)→phrase(2小節)→period(4小節=問い/答え)`。各句末に**休符 or 長音（息継ぎ）**、終止で**安定音(1/3/5・period末は主音)へ着地**、**頂点を後半≈0.62**に1つ。＝WuYun/GTTM の「骨格を先に作り装飾で埋める」を**学習なし・記号的に**模す。
2. **決定的・ルール＋制約・既存資産を壊さない**：#86 どおり LLM は音符に触らない。手法は **CSP的規則（音域/跳躍上限/跳躍後は順次反行/輪郭ターゲット/句末closure）＋モチーフ変換（反復/模続/反転/拡大縮小/断片化）**。純マルコフ単独（無限に続く・コーパス必須）とニューラル（重い・非決定）は**採らない**（知見源に留める）。コーパスがあれば**弱マルコフを"制約を満たす範囲のバイアス"**として後付け（無くても規則だけで回る＝degrade gracefully）。
3. **骨格＝エッセンス（連想記憶との統一）**：生成の「骨格」と、参照メロから抽出する「エッセンス」は**同じ表現**。参照メロを骨格抽出（GTTM time-span 還元の軽量版）→**別の進行/別の肉付けで再構築**＝「似て非なるメロ」。生成と連想が1つの骨格概念で繋がる。
4. **著作権はコードより一段慎重**（メロは最も保護される層）：エッセンスは**抽象/相対/統計の層のみ**保持（音程列・輪郭・リズム指紋・度数/PC分布・非和声音の使い方）。**絶対ピッチ＋絶対リズムの生メロ全曲は保持/配布しない**。生成出力は**元曲との最長共通一致(LCS)を上限制約**で断つ。「度数列＝権利対象外」の整理はメロでも**抽象層なら通る見込み**だが運用で担保。

## 2. 推奨アーキテクチャ（3層・決定的）
```
[骨格層] phrase/period を割り当て→句末に休符＋安定音着地→頂点(≈0.62)を配置（呼吸を先に作る）
   ↓
[制約層] 骨格間を装飾で埋める：コードトーン拘束(既存)＋CSP規則(音域/跳躍/跳躍後反行/非和声音は順次解決)
   ↓
[変奏層] 既存のモチーフ反復＋変換(模続/反転/拡大縮小/断片化)を**句機能で位置駆動**に（consequent=模続、句末=拡大/断片化）
```
既存 `genMelody`（リズム図形＋度数コントゥア＋モチーフ反復）は**変奏層そのもの**＝捨てずに上に骨格層を載せるのが最小改修・最大効果。

## 3. エッセンス表現（連想記憶の材料・著作権セーフ度つき）
層を連結した「メロ指紋」で運用（コード進行の指紋ベクトルと同型）。
- **E1 輪郭(Parsons/5段階)**：身振り。最も安全・識別力低。
- **E2 音程列(移調不変)**：旋律骨格・"らしさ"の主力。**既存 `melodySimilarity` がこれ**。高安全。
- **E3 リズム指紋(IOI比/拍位置/シンコペ率)**：ノリ。高安全。←今の類似度が無視してる重要層。
- **E4 度数/PC分布**：旋法・多用する度数。高安全。
- **E5 非和声音の使い方(経過/刺繍/掛留/アポジャトゥーラの癖)**：個性の核。**既存 `analyze_fit` が材料**。高安全。
- **E7 フレーズ構造(反復/対句・自己相似)**：骨組み。SIATEC/COSIATEC or n-gram反復検出。
- **E8 絶対ピッチ＋絶対リズムの同時一致**＝**複製＝使わない/避ける対象**（コピー検出の上限）。
「らしさ」＝統計/相対の層（E2/E3/E4/E5）。固有表現＝E8（具体の長い一致）。

## 4. エッセンス→"違うメロ"生成（似て非なるの作り方）
**factorize & recombine**（保持する層／作り直す層を選ぶ）：
- G1 輪郭+リズム保持・音程作り直し（フレーズの寄せ）
- G2 リズム保持・ピッチ再サンプル
- **G3 統計(E2/E4/E5)を種に新規**（"癖"だけ継ぐ＝最も連想らしい）＋**G4 モチーフ変換で展開**（似て非なるの古典解）
いずれも**コードトーン拘束＋analyze_fit 適合**で合法な音に閉じる。
**境界＝二重しきい値**：`θ_lo ≤ 参照との類似 ≤ θ_hi`（下限=エッセンス保持／上限=コピー回避）＋**絶対ピッチ＋リズムの LCS が n音以上一致したら棄却**（"originality report" 方式・法的にも安全側）。決定的・TDD固定可。

## 5. メロコーパスの現実解
- **Hooktheory/TheoryTab(HLSD)型**＝U-FRET 取込の完全な対応物：**調基準の度数で human-transcribed の melody+chord・注釈メタデータのみ配布（音源は配らない）**。法的整理がコードと同型。J-POP薄→自前で度数化投入。
- 検証用に **Essen/民謡（権利クリーン）**。正規化＝調推定→度数列＋音程列、各音にコード文脈＋非和声音分類、多粒度索引（全体/フレーズ/2小節動機/(動機,コード)）。

## 6. 段階導入（縦スライス・各段は前段を壊さず property を足す）
- **S1（最優先・不満の核を直す）**：骨格層＝phrase/period＋**句末に休符/長音＋安定音着地**。既存反復・コードトーン拘束は維持。→「呼吸しない」が解消見込み。
- **S2**：後半≈0.62 頂点アーチ＋跳躍後の順次反行。
- **S3**：句機能で位置駆動の変奏（consequent=模続／句末=拡大・断片化）＋非和声音の役割化（順次解決保証）。
- **S4（連想の入口）**：骨格/エッセンス抽出＋**多層 melodySimilarity（音程＋リズム＋輪郭）**＋「このメロに似てる/かぶってない？」retrieval（メロ版 M2）。embeddings不要・既存DPの発展。
- **S5（北極星）**：エッセンス→違うメロ生成（G3/G4＋二重しきい値＋LCS上限）＋メロコーパス整備＋様式/アーティスト指紋。

## 7. 受け入れ基準（property test 案・既存 generate.test の流儀＝seed固定で musical ルールを担保）
1. 各 phrase 末に**休符 or 規定長以上の長音**が必ず存在（全曲が onset で埋まらない・period末≥phrase末）。
2. 終止が安定音着地：consequent/period 末音=主音(コードI上)、antecedent 末∈{属音,中音}。
3. 最高音(頂点)の位置 `start/total ∈ [0.5,0.75]`・頂点1〜2回・頂点後の平均ピッチ<前。
4. モチーフ反復が認識可能（dur列/コントゥア列が小節間で高一致・既存テスト拡張）。
5. 4度以上跳躍の直後は逆向き順次(≤4半音)が高割合・総音域≲1.5oct。
6. 非和声音は順次で入り順次で解決（孤立跳躍非和声音ゼロ・掛留は下行解決）。
7. mood連動：sparse の句末無音 > busy（既存「busy>sparse 音数」維持）。
8. 決定性：同 seed で完全一致。
9. （S4/S5）抽出骨格↔再生成メロ＝**エッセンス保持かつ表層は別物**／コピーゲートが LCS≥n を棄却。

## 8. 未確定・要ユーザー判断

- S4/S5 の優先度（呼吸＝S1-3 を先に仕上げてから連想 S4-5 か、早めに連想の縦スライスも）。
