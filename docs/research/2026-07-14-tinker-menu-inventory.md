## 2. 煩雑さの定量

### 2.1 縦スクロール総量（モバイル390×844・シート可視高 691px）

- **詳細畳み時**：scrollHeight **935px**（生成11＋プリセット帯＋メロ加工4＋書き出し2）。可視691px＝**244px が初期ビューで隠れる**。
- **詳細展開時**：scrollHeight **1756px**。可視691px＝**1065px スクロール必須**＝**画面 約2.5枚分**を上下に繰る。
- 途中に **5つの群見出し＋3セクション区切り**が挟まるが、全ノブが同格の帯として縦一列に積まれる（`flex-wrap:nowrap`固定・2026-07-10 doc §1.1 が指摘した「器の限界」がそのまま残存）。

### 2.2 主要操作までのタップ到達深さ

| 操作 | タップ列 | 深さ | スクロール |
|---|---|---|---|
| デフォルトでメロ生成 | いじる → メロ | **2** | なし |
| プリセットでメロ生成 | いじる → プリセットchip → メロ | **3** | なし（プリセット帯は畳み時も見える） |
| ドラムを型指定で生成 | いじる → ビート型Sel選択 → ドラム | **3**（Sel操作込みで実質4アクション） | なし |
| 詳細ノブを1本回す | いじる → 細かく設定する → （最大1065pxスクロール）→ ノブ | **3**＋長スクロール | 群による |
| 「人間味/ベースをよける」（最下部群）に触る | いじる → 細かく設定 → **1400px超スクロール** → seg | 3＋最深スクロール | 最大 |
| 生成物を置く | （生成後）候補カード「置く」 | +1 | トレイ位置まで |

到達深さ自体は浅い（2〜3タップ）が、**詳細段の「縦1756pxの壁」＋全38コントロールが1シートに同居**が煩雑さの実体。プリセットで畳めるが、ノブ・セレクタ・生成ボタン・書き出しが**器を分けずに一列**に載る。

## 5. 再設計への含意（材料メモ・提案はFableへ委譲）

- **煩雑さの芯＝「器」でなく「1器への同居量」**。ボトムシート化(P3)は済んだのに、38コントロール・縦1756pxが1シートに載る。3docの共通眼目「中身を減らす」が未達。
- **今日のWP群が煩雑さを加速**：ドラム/ベースのジャンル型セレクタ4本（42選択肢）が生成ボタン列に割り込み、生成の主動線（メロ→ベース→ドラム）が長selectで分断（スクショ02-03）。**"型を選ぶ"は生成ボタンとは別レイヤーに逃がす余地**。
- **前面/詳細の二層が中途**：param-clarity/consolidation が設計した「前面4本常時＋残りは詳細」が実装では「全ノブ詳細／前面はプリセット＋🎲」に化けている。**プリセットで足りる人**と**ノブを回す人**の分岐は効いているが、前面に何も残らない＝畳み時のシートが「生成ボタン＋型セレクタ＋プリセット＋加工＋書き出し」の雑多な縦列になっている。
- **群は5つあるのに視覚の階層が弱い**：群見出しはあるが全ノブ同幅の帯で、優先度・文脈依存（④は在時のみ）が地の文に埋もれる。
- **生成/加工/書き出しが1シート同居**：本来スコープの違う3系統（作る／メロを直す／出力）が縦に連続。器の分割余地。
