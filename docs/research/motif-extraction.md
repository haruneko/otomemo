# モチーフ：外部調査＋実曲からの抽出アプローチ設計

目的：**実曲からモチーフを抽出・分析できれば、生成の目処が立つ**（モチーフ＝メロの「胚」。これを取れれば、再利用＋展開で motivic な生成ができる）。最終更新 2026-06-28。

## 1. モチーフとは（音楽理論）
- **定義**：作品の中で特別な重要性をもつ/特徴的な、**短い音型(数音)**＝メロの最小の意味単位。**リズム＋音高(音程)の組**。salient で recurring。
- **展開技法（motivic development）**＝モチーフを変えて統一感を出す：
  - **repetition** 反復（そのまま）
  - **sequence** 反復進行（音程関係を保って移調）← pop で最重要
  - **inversion** 反行（上下反転）
  - **retrograde** 逆行
  - **augmentation/diminution** 拡大/縮小（リズムの伸縮）
  - **fragmentation** 断片化（一部だけ取り出して展開）
  - **extension/truncation** 延長/切詰
- ＝メロは**1〜数個のモチーフを展開して**作られる。だから「モチーフを使ってる感じ」＝**同じ音型が(変形しつつ)戻ってくる**こと。

## 2. 計算機による抽出（MIR：motif / repeated-pattern discovery）
- **SIATEC / COSIATEC**（Meredith）＝幾何的パターン発見。音符を多次元点とし、**平行移動(translation)で一致するパターン群(TEC)**＝反復＝を全列挙。移調・時間ずれに強い。
  - 弱点：**TECが数万出る＝知覚的に有意なのはごく一部**＝salience でフィルタ要（COSIATEC/SIATECCompress で改善）。
- **string/n-gram系**：メロを **(音程, 音価)** 列に符号化→部分列の頻度（suffix tree / n-gram）で反復を見つける。音程ベース＝**移調不変**。
- **self-similarity matrix**：音程列の自己相似行列→**対角の縞=反復箇所**。
- ＝鍵は **(a)音程+リズムで符号化(移調・テンポ不変) (b)反復を見つける (c)salienceで絞る**。

出典: [Motif (music)–Wikipedia](https://en.wikipedia.org/wiki/Motif_(music)) / [Motif development–Vaia](https://www.vaia.com/en-us/explanations/music/music-composition/motif-development/) / [SIATEC: Algorithms for Discovering Repeated Patterns (Meredith)](http://www.titanmusic.com/papers/public/siajnmr_submit_2.pdf) / [BPS-Motif dataset, ISMIR2023](https://archives.ismir.net/ismir2023/paper/000032.pdf) / [Improving Motif Discovery, TISMIR](https://transactions.ismir.net/articles/10.5334/tismir.250)

## 3. 我々の抽出アプローチ（POP909・単旋律向け＝軽量に）
SIATECは重い/出すぎ。我々は**単旋律＋8小節**なので軽量 string/self-similarity で十分。

### 表現（移調・テンポ不変）
- 各音を **(scale-step音程, 音価カテゴリ)** に符号化。音程＝**音階度数の差**（移調不変）。音価＝{16,8,4分…}。
- メロ＝この記号列。

### 抽出手順
1. **窓**＝2〜6音 or 1-2小節を全位置でスライド。
2. **類似**＝2窓が「音程列が一致(±許容) かつ リズム列が一致」＝同一モチーフの出現（移調OK＝sequence、反転OK＝inversion を別フラグで）。
3. **主モチーフ**＝出現回数×長さ が最大の窓（salience）。1曲から1〜数個。
4. **展開タグ**＝各出現が exact / transposed(何度) / inverted / rhythm-varied のどれかを記録。

### 出力＝「モチーフ辞書＋展開統計」
- モチーフ図形（音程+リズム）の**語彙**（実曲群から）。
- **長さ分布**（何音/何拍）・**1曲あたり何回出るか**・**展開の内訳**（反復/sequence/inversion/拡縮 の割合）。
- ＝「実曲のモチーフはこういう図形・これくらい跳ぶ・これくらいの頻度で・こう展開する」が数値で出る。

## 4. 抽出→生成のパス（なぜ分析したいか）
分析できれば生成が決まる：
- **語彙**：抽出したモチーフ図形（跳躍/3度を含む実物）を**生成のコミット対象**に＝今の「歩くだけ」を実図形に置換。
- **展開**：抽出した展開統計（sequence○%/inversion○%…）どおりに**モチーフを展開**＝「ちゃんとモチーフを使ってる感じ」。
- **骨格との関係**：モチーフの出現位置 vs 骨格点／コードを分析＝モチーフをどう移調して harmony に合わせるか（sequence の移調量）を学習。
- ＝**「歩く skeleton 追従」→「実モチーフをコミットして実統計で展開」**＝motif感・跳躍・脱ダルダルが一気に入る。

## 4.5 ★抽出プロト実測（POP909 201曲・8小節）＝生成の目標値
| 指標 | 実測 | 含意 |
|---|---|---|
| 主モチーフ長 | **平均3.6音**（4音40%/5音13%/6音23%） | モチーフ＝**短い4音前後**の図形 |
| 出現回数 | **2.4回/8小節** | 数回だけ戻る（句を毎回コピー＝やり過ぎだった） |
| メロ占有率 | **23%** | **モチーフは1/4だけ**。残り77%は展開＋自由材料 |
| モチーフ内跳躍 | 5% | exact反復する核は順次寄り |
| 反行を使う曲 | 20% | inversion は2割で展開に使う |
| リズム変形 | 12% | augmentation/diminution は1割 |

### ＝重要な構造の発見（今までの失敗の原因）
- 実曲＝**「短い4音モチーフ(23%・認識できる) ＋ 自由材料77%(展開・跳躍はここ)」**。
- 我々の失敗：①**句を丸ごとコピー**＝反復しすぎ(実際は2.4回・23%だけ) ②**全部を順次diminution**＝自由材料77%に跳躍/3度が無い＝ダルダル。
- ＝**モチーフは“短く・数回・1/4だけ”、残りは自由に跳ぶ**。これがmotif感＋脱ダルダルの正解。

### ＝refined 生成ターゲット
1. **短い4音モチーフをコミット**（跳躍は少なめでも可・リズムが特徴的）。
2. **2.4回/8小節だけ**戻す（反行20%・リズム変形12%を混ぜる）＝認識できるが くどくない。
3. **残り77%は自由材料**＝move学習(跳躍14%/3度18%含む)で動く＝ダルダル解消。
4. 骨格＝全体の構造、モチーフ＝前景の hook。両立。

## 5. 実装の段取り
1. **抽出器プロト**：POP909メロを(音程,音価)符号化→self-similarity/窓で主モチーフ＋展開タグ抽出。**統計を出す**（長さ・頻度・跳躍率・展開内訳）。← まずここ（分析）
2. **モチーフ辞書をDB/同梱**化（骨格モデルと同様）。
3. **生成を motif-driven に**：辞書からモチーフ選択→展開(sequence/inversion)で8小節を組む→骨格/harmonyで移調補正→diminutionで隙間。
4. 評価＝抽出時の統計（跳躍率/展開内訳/同音率）に生成を合わせる（FMDでなく）。
