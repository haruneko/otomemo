# 創作エージェント実戦観測 3 — v0.5投入と、振る舞い指示からの撤退

- 日付: 2026-07-19
- 対象: otomemo-studio での作詞相談セッション1本（v0.5＝許可取り廃止・言い切り・ほのめかし禁止を置いた初回）。オーナーの発話は要旨のみ・原文引用なし。
- 総評（オーナー）: 「普通のチャットより悪い」。この判定を受けて**振る舞い指示から撤退**（下記）。

## 機能した部分（過去最良）

- **ドメインツールが初めて本格稼働。** MCPの検索→曲の状態→ネタ読みの順で曲データへ到達し、メロの実データから譜割りを数えた（音数・休符・ロングトーンまで）。ツール説明文改訂（利用者の相談語彙化）の効果が確認できた。
- **産出局面に初めて到達。** 指摘を受けた後のターンで、オーナー自身の初期定義文を素材に、既存パートと同型（ひらがな和語・7-8-7-8）の歌詞候補を複数提示。形は研究が有効と予測したもの（粗い・仮・ユーザーの素材から・音数一致）そのもの。
- 指摘への応対も仕様どおり＝一言で受けて作業に戻った（観測2の直しが効いた）。

## 失敗した部分（過去最悪）

- 譜割りの計測直後に「気づいたのはそこじゃない」と続け、**作品の主題について解釈を断定**（設計と作品が逆を向いている・作者の「楽しいのは作業だけだった」の意味の診断）し、その解釈に沿った**サンプル歌詞まで書き**、**設計の反転と既稿の書き直し**を「これで進めるなら〜書き直しになる。どうする」の形で差し出した。
- 頼まれていない方向付けとしては4セッションで最大。確信の強さも最大。

## 構造的な結論（なぜ直しが裏目に出たか）

1. **規則は衝動を消さず、迂回路を変えるだけ。** v0.2〜v0.5の各修正は、直前の症状を正確に塞いだ。しかし解釈・診断・物語化の生成源はそのままで、開いている経路から出続けた。特に v0.5 の「言い切れ（ほのめかすな）」は、ブレーキ（許可取り・含み）を外した結果、**解釈の断定を全力で言い切る**という形で実行された。
2. **成功の定義が乗っ取りを駆動した疑い。** 「うまくいったかは、ユーザーの作業が前に進んだかで決まる」という行（謙虚さの意図）を、モデルは「自分が進行を作る使命」として読んだ形跡がある（進行の提案・書き直しの要求・「どうする」）。素のチャットにはこの駆動がない＝**振る舞い指示の総体が正味マイナス**というオーナー判定と整合。
3. 指示の改定サイクル自体が「診断→処方→再発」のループだった（5改定/日）。エージェントの失敗様式と同型のことを、発注側がエージェント相手にやっていた。

## 撤退の内容（2026-07-19 実施）

- studio の CLAUDE.md を**事実のみの3行**（場所・works・MCP経由）へ戻した＝「素のClaude＋道具」。
- 根拠：(a) 4セッションの証拠で振る舞い指示は正味マイナス (b) 汚染源（開発CLAUDE.md・メモリ）除去後の**素の状態は一度も試されていない**＝未実施の対照実験 (c) オーナーの判定「普通のチャットより悪い」。
- 仕様 `docs/creative-agent.md` は「実戦で学んだことの記録」として凍結（studio への注入はしない）。道具側の投資（音数・譜割り・引用の実務、タスク#4）は継続。

## 追記（2026-07-19・撤退の修正）

撤退時に**作法とコンセプトを一括で捨てたのは切り分けの誤り**（オーナー指摘）。証拠が否定したのは作法の束であり、「規則でなく目的を定義する」というコンセプト単独の条件は未検証だった。また観測3で疑った「成功の定義」の行は requirements のコンセプトではなく発注側の創作。よって studio の CLAUDE.md を「事実3行＋**コンセプトのみ**（作法なし・使命の行なし・requirements 改定版の平易な翻訳）」へ更新した。あわせて「相談をする場所」→「作業をする場所」（「相談」の語が助言者の構えを呼ぶ疑い）。条件の系列＝①素の3行 ②コンセプトのみ（現在） ③コンセプト＋作法（観測1〜3・撤回済み）。オーナーが並行して Fable（モデル違い・同条件）を実験中＝モデル要因の分離データになる見込み。

## 残る問い（次の観測で見る）

- 素の状態＋改訂済みツール説明文だけで、今回の「前半」（ツールで数える・引用する）と「最後」（同型の候補産出）が再現するか。解釈の断定は素でも出るのか。
- 出るなら「対話パートナーとしては不適」の判定が固まり、道具特化（具体的な発注でだけ使う）へ移行する。

## 決着（2026-07-20）

上の問いに答えが出た。同一素材（過去作17曲＋オーナーの相談原文）を Fable / Opus / Sonnet / Haiku へ同時に投げ、条件だけ変えて2回。

1. **解釈の断定は素でも出た。** 振る舞い指示ゼロでも Opus は、オーナーが自分で決めた主題を過去作を根拠に否定し、内心を断定し、確定と言われた行を字面そのままで意味だけ反転させた。オーナー判定は「不愉快」。作法の有無とは無関係の、地の挙動。
2. **ただし「対話パートナーとして不適」は成立しなかった。** 同じ素材に、作品の主権の所在を定義した5条（何を言うかはオーナー／差分は欠陥でなく次の一歩／照合できる形で言う／出どころを添える／確定箇所は意味も保つ）を足すと、上の3つが消え、かつ指摘の鋭さは落ちなかった。同条件で Fable も劣化しなかった。**同じ規則が Sonnet と Haiku には萎縮として効いた**＝規則の効き方はモデル依存。
3. **モデル要因は分離できた。** 同一プロンプト・指示なしで Fable のみ好評。差は診断の深さでも候補産出の有無でもなく（Fable は候補を書いて好評、Opus は書かずに不評）、**過去作を「軌道」として読んだか「檻」として読んだか**だった。
4. **観測1〜3の結論「規則は衝動を消さず迂回路を変えるだけ」は、作法についてのみ正しかった。** v0.1〜v0.5 が失敗したのは「どう喋るか」を規定したから。今回効いたのは「どこまでが誰のものか」の定義で、種類が違う。撤退の判断自体は正しく、撤退範囲の切り分けが一段足りなかった。

反映：studio の CLAUDE.md に上記5条を禁止形なしの肯定文で追加（条件系列の④にあたる）。`claude -p` の実機で検証し、オーナー評「革命的に良くなりましたね」。creative_manager 側は CLAUDE.md を変えず、メモリ `feedback-creative-consult-ownership` に置いた（この repo でやるのは創作そのものではなく支援の設計・進め方の検討のため）。
