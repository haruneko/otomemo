# 骨格メロの構造使い回し＝フォーム型リテラル回帰（研究＋設計・2026-07-13）

**動機（オーナーFB）**：骨格メロに反復が乏しく「8小節ランダムウォークに見える」。「2/4/8で構造を使い回せないか」。
→ 現状の反復機構を実測で押さえ、**階層的・リテラルな回帰**（AABA/period 等・"同じフレーズが返ってくる"）を設計する。

関連：[[2026-07-11-melody-phrasing]] 系（句フレージング）・[2026-07-10-motivic-repeated-note-melody](2026-07-10-motivic-repeated-note-melody.md)（反復音モチーフ）・
design #20（骨格層）／#12-M（メロ生成）。本 doc は**骨格の構造線**（`genSkeletonFromModel`）に閉じる（表面のモチーフ反復は別層）。

## 1. 現状＝何が既にあるか（実コード・`apps/api/src/music/melodyCells.ts:160-223`）
`genSkeletonFromModel` は**2小節ユニットの動機反復**を既に持つ：
- `spu = strongQ.length*2 = 4` スロット/ユニット（強拍 [0,2]×2小節）。`nu = ceil(bars/2)`。8小節→`nu=4`。
- 反復（L203）：`reuse = u%4===1 ? hA : u%4===3 ? hB : null`。`hA`=u0 の輪郭（連続ステップ移動）を記録（L216）、`hB`=u2（L217）。
  → **A A′ B B′**（u0 fresh→u1=Aの反復、u2 fresh→u3=Bの反復）。反復強度 `rep=repetition??0.85`（L193・実曲54-60%対応）。
- 反復の中身（L210-211）：ユニット**頭スロット s∈[1, spu-2]** のみ「前音 `pi` から `reuse[s]` ぶんステップ移動」を `rep` 確率で反復。
  s=0 は fresh（`idxOf(ctr…)`）、s=spu-1 はカデンツ（句末 5̂／曲末 1̂）。

## 2. なぜ「ランダムウォーク」に見えるか（診断・3点）
1. **反復が輪郭(ステップ移動)で、直前音 `pi` にアンカー**（L211 `cl(pi + reuse[s])`）。`pi` はユニット跨ぎでドリフトするので、
   同じ"身振り"は繰り返すが**同じ絶対音には戻らない**＝耳/目に反復として届きにくい（裸の骨格は他の同一性手掛りが無く効きが弱い）。
2. **再利用は頭スロットだけ**（s=0 fresh・s=spu-1 カデンツ＝実質 spu-2=2 スロットのみ）＝1ユニット4点中2点しか揃わない。
3. **後半 B B′ は新規素材**＝**8小節スケールでの A の回帰が無い**。AABA のような「戻ってくる」構造が欠落＝全体がさまよう。

## 3. 狙い＝階層的・リテラルな回帰（音楽形式）
実メロの安定感は**素材が複数スケールで返る**ことから来る：動機(2小節)→句(4小節)→楽節/period(8小節)。
- **period（8小節 [4+4]）**：後半4小節が前半4小節の**リテラル反復**、句末カデンツだけ変える（前半=半終止HC上の 5̂／後半=完全終止PAC上の 1̂）。
- **AABA**：A が u0・u1・u3 に**リテラル回帰**、B(u2)だけ対比。J-POP/歌ものの王道。
- リテラル＝**絶対度数インデックスをそのまま複写**（輪郭でなく）＝耳に「同じフレーズ」。カデンツのみ差し替え＝終止の役割は保つ。

## 4. 設計＝`skelForm` opt-in ノブ（既定 undefined＝現状 A A′ B B′＝bit一致）
`genSkeletonFromModel opts.skelForm?: "period" | "aaba"` を追加。**フォーム計画** `formPlan(form, nu)` が各ユニット u に
「複写元ユニット `src[u]`（null=fresh）」を与える：
- `period`：`src[u] = u < nu/2 ? null : u - nu/2`（後半＝前半の複写）。nu 奇数は末尾 fresh でフォールバック。
- `aaba`：4ユニット周期で `[null,0,null,0]`（u1/u3 が u0 を複写・u2 は fresh）。nu=4→AABA、nu=8→AABA×2。
- 複写の実体：`src[u]!=null` なら当該ユニットの全スロットで `I[base+s] = I[srcBase+s]`（**リテラル**）、
  ただし**カデンツスロット（s=spu-1）は既存の句末ルール**（`phraseEnds`/`u%2`/曲末）で上書き＝終止の役割は形式で保つ。
- 既存の輪郭反復（hA/hB）は `skelForm` 未指定時のみ（＝現状路・bit一致）。指定時はリテラル複写が優先。

**なぜ genSkeletonFromModel に置くか**：骨格＝構造線の SSOT。表面（`genMotifMelodyV2`）は骨格を句頭アンカーに使うので、
骨格が形式的に回帰すれば**表面も自然に回帰**する（骨格→表面の一方向依存・design #20）。骨格1か所で階層反復が効く。

## 5. bit・耳・段取り
- **bit**：`skelForm` 未指定＝完全 bit一致（既存 golden 不変）。指定時は当然変わる（新規挙動＝新テストで固定）。
- **耳較正（残・音が要る）**：period/aaba の"効き"、リテラル回帰が単調過ぎないか（B の対比量・カデンツ差の大きさ）。
  裸の骨格で強く出す（rep と同様、骨格は同一性手掛りが少ない）。
- **実装スライス**：
  - S1（本 doc の核）：`genSkeletonFromModel` に `skelForm`＋`formPlan`＋リテラル複写。TDD＝既定bit一致／period で後半=前半（カデンツ除く）／aaba で u3≈u0。
  - S2：`gen_skeleton`（generate.ts:563-596 の候補出し）＋ MCP スキーマに `form` を透過。web（机/SectionEditor のノブ）露出。
  - S3：耳較正（音が聴ける時）＝既定値・テンプレ追加（ABAB/AABB）判断。
- **スコープ外**：表面レイヤのモチーフ保存（motifMode:preserve 既存）とは独立。今回は構造線のみ。

## 6. 決定事項（design へ）
- 骨格に**フォーム型リテラル回帰**を opt-in で足す（既定=現状）。正典＝design #20 に追記。実装は S1 から。
