# 案B 動機保存レンダ＝アーキ設計spec（v2・二眼審査反映）

> 状態＝**設計spec v2・実装前**。v1をコード眼(Opus)＋理論眼(Fable)の二眼審査（両者「骨子GO・記述要修正」）で得た**コード5点＋理論6点**を反映。親research＝[[2026-07-10-motivic-repeated-note-melody]]。全て既定＝bit一致の新経路 `motifMode:"preserve"`（未指定＝従来レンダ）＋独立ノブ hook/articulation/inflect（既定0）。

## 0. 設計目標（合格バー）

1. **動機の同一性が保存される**：提示動機の**リズム＋輪郭（方向列）**がA/A''反復・移高反復で保たれる（同一性の担い手は輪郭＝Dowling 1978・厳密音程は二次）。オーナー例 `A A B C A / B G G` を seedMotif で渡すと提示ブロックで**実音輪郭が移高を法として再現**（度数でなく実音で測る＝理論#1）。
2. **和声適応は「音を曲げる」でなく「置き場所を選ぶ」**：動機内部の度数差分は不変、動機全体の移高段kを総コスト最小で選ぶ。deg空間の+k移高＝**tonal transposition**（半音実質は移高先で変わる＝仕様であって同一性破壊ではない・理論論点1）。
3. **反復音が一級**：move=0が動機の骨として保持。ただし**位置依存**（句頭アンカー/句末クローズで高く・中間で低く）＋**連続同度上限2〜3音**（一律確率は「フック」でなく「どもり」を作る＝理論#4）。
4. **既定bit一致**：`motifMode`/hook/articulation/inflect 未指定＝従来レンダ＝既存出力と deep-equal。
5. **ガードレール維持**：禁則0・音域維持・決定性。単一頂点は保護優先（反復音フックは足場型＝複数平頂点が正常）。**カデンツは動機保護に優先**（歌える最低条件＝理論#2）。

## 1. U1 動機オブジェクトモデル（deg は**導出**・格納しない）

現 `Motif16 = { ons, mv, run }`（mv＝半音差分）を**source-of-truthとして温存**。degは格納フィールドにしない＝**preserveレンダの実現点でのみ `deg = cumsum(mv)`（deg[0]=0＝開始音アンカー）を遅延導出**（コード#2＝二重表現の同期崩壊を構造的に回避。varyTail/invert/fragment/applyConverse/freeVary は既存mv演算のまま無改修で自動整合）。

追加するのは**発音と表情**のみ（動機生成には触れない）：
```ts
interface MotifArt { art: Articulation[]; accent?: number[]; } // ons と同長・onset毎
type Articulation = "legato" | "staccato" | "restAfter";
// realize時: deg = cumsum(mv), deg[0]=0（開始音基準に確定＝invert が単純に deg→-deg・code#2の baseline二択を解消）
```
- **accent 既定**：連続同度グループの頭は既定でアクセント（連打は micropause だけでなく連打頭アクセントで立つ＝理論足す#5・Bresin）。`accent?` は任意だが連打グループには既定パターンを与える。
- **短調の導音/色音**：deg=構造／実現時にCT snapで半音を許す（可変6/7度＝Aldwell&Schachter の短調文脈変化形と同型。変化形の違いは同一性を壊さない＝理論論点5）。ただし snap は§3の不変量に従う。

## 2. U2 hookノブ＋保護マスク（案A入口）

`hook: 0..1`（既定0）。

**bit一致の要＝単一ドロー形**（コード#1・v1のpseudocodeは r()を2回消費して壊れる）。`mkMotif :625`／`varyTail :686` を：
```ts
if (m === 0) { const u = r(); m = (hook > 0 && u < hookKeep(pos, hook)) ? 0 : (u < 0.5 ? 1 : -1); }
// hook=0 → hookKeep=0 → u<0 常偽 → u<0.5?1:-1 ＝原と完全一致・r() 1回消費＝bit一致
```
- **hookKeep は位置重み付き**（理論#4）：`hookKeep(pos, hook)` は動機頭/末で高く中間で低い。**連続同度は上限2〜3音**（超えたら強制的に±1へ）。
- `render`（現`:761`反同音）：**動機由来（deg[i]===deg[i-1]の意図的同度）は素通し**、snap衝突由来の偶発同音のみ従来散らす。区別は導出deg参照。※hook=0では mvに0が無く素通しブランチは死にコード＝bit一致自明（コード眼確認）。
- **保護マスク** `motifProtected:Set<number>`：§4。

## 3. U3 動機保存レンダ（本丸・別関数 `renderPreserve()`）

コード#5＝inline分岐でなく**別関数 `renderPreserve()` 新設**。挿入点は**2箇所**：k選択＝ブロック組立ループ級（`:864-884`付近）／realize＝音級。default は本関数に一切入らない＝§6を構造保証。

### 3.1 移高kの選択（ブロック級・決定的）
候補 k∈[-4,+4]段からコスト最小を採る。deg内部差分はk不変＝形が保たれる。
```
cost(k) = w0·(1 - カデンツ着地(M,k))          // 最重量＝カデンツ強拍に終音が乗るk（理論#2）
        + w1·強拍CT質ペナルティ(M,k)            // "率"でなく質（下記・理論#5）
        + w2·対ベース衝突(M,k)                  // b9・並行完全（既存 counter 資産流用）
        + w3·音域はみ出し(M,k)
        + w4·前ブロック終端からの跳躍(M,k)      // 動機間接続
        + w5·|k - kPrevSameLabel|               // ヒステリシス＝同ラベル(A/A'')は同一k優先（理論#6・M10自己相関）
```
- **骨格kの役割分担**（コード#4）：`baseIdx = blockAnchorFromSkeleton`（粗いレジスタ＝既存 :795,:878）、**kは和声微調整のデルタ**。役割リフト(role===2 の+2 :879・seq +2 :873)は baseIdx側に残し、kと二重制御しない。`pitch[i] = clampScale(L, baseIdx + k + deg[i])`。
- **強拍CT質ペナルティ**（理論#5＝裸の率0.6を廃止）：強拍非CTを許すのは (a) CTから2度以内（倚音/刺繍＝順次解決可能）、(b) b9衝突なし（メジャー上の完全4度等＝avoid note・Levine）。add9/6/maj7系の非CTは実質協和ゆえペナルティ対象外。**カデンツ強拍CTは常に100%**。**ペダル端点規則**：反復音アンカーが複数ブロックに跨るとき、最初と最後の強拍出現がCTなら中間の非CTは許容（One Note Samba のペダル論理）。※端点検出（跨ブロック追跡）は最重量ゆえ**未実装時は厳格側（＝ペダル免除なし＝全強拍を質条件で評価）へ安全に縮退**（再審査#3・ブロッカーでない）。
- **単調化ガード**（理論#6）：k≠0の連鎖（真のsequence）は seq役割ブロックに限定・連鎖2〜3段まで・移高幅は2/3度中心。将来検討＝k列全体のDP最適化（線が先を見て歌う）。v1は貪欲＋ヒステリシスw5。

### 3.2 実現（realize・音級）＝同一性不変量（理論#1・最重要）
選んだkで各音を実現する際、動機の同一性を壊すsnapを禁ずる：
- **(a) 同degグループは一括で同一pitchに実現**（snapはグループ単位＝ラーラを半音違いのどもりにしない）。
- **(b) snapは隣接moveの符号（上行/下行/同度）を変えてはならない**（輪郭優位・Dowling）。
- 満たせない強拍snapは行わず、§3.3のリトライへ。

### 3.3 適応フォールバックの階段（理論足す#1＋コード#5訂正）
kが配置衝突（対ベース/音域/CT）を解けないとき、**動機を全崩壊させる前に段階的に緩める**：
1. **inflect ノブ（0..1・既定0）**：動機**末尾1音のみ ±1段**の変更を許す（tonal answer・varyTailと同じ「末尾は可変・頭部のリズム＋輪郭が同一性を担う」＝理論足す#1）。
2. **truncation**：カデンツ前で動機を切り詰め、終音をカデンツ着地に（切詰は標準展開技法＝motif-extraction §1・理論#2）。
3. **従来snapフォールバック**（最終手段＝動機を諦める）。
- **内部禁則の扱い**（コード#5訂正）：動機内部の三全音/7度は**k不変で全kに現れる**＝kやり直しでは解けない → **inflect（末尾）で解けなければフォールバック確定**。「全k不合格→従来snap」はこの内部禁則ケースのみ。kが解けるのは配置衝突（動機×ベース/音域/CT）だけ、と分類を訂正。
- **inflectの適用範囲（再審査の締め）**：inflectは**末尾1音専用**。動機**内部**の強拍配置衝突をkが解けない場合はinflectでは救えず直接snapへ（内部はkが担当・末尾はinflectが担当、の分担）。

## 4. U4 保護契約

- **構築タイミング**（コード#3a）：`motifProtected` は render時indexでなく、`notes.sort`（:885）＋breatheの**notes再構築（:903）後**に、動機構成音の**start時刻/ブロック帰属から後処理直前に再判定**（cadenceIdx :1066 と同流儀）。render時index捕獲は不可。
- **保護ガードの設置先（直接上書きパスも明示・再審査#2）**：`if(protected.has(i)) continue` を置くのは (a) 強拍CT+anti-unison `:974-1004`（最も保護音を潰しやすい・必須）、(b) 単一頂点 `:1040-1057`、(c) 弱拍濁り掃除 `:1139-1166`（`:1159`）の3直接パス冒頭。
- **隣接書込先も保護**（コード#3b＝lockedより厳密）：後処理は隣接音も動かす（fixForbidden i-1 `:1014/1018`・gapFill i+1 `:1029`・カデンツ li-1 `:1088`）。隣接書込は**全て placeNonForbidden/placeNear 経由**なので、この2ヘルパで「書込先indexが保護ならskip」を1点判定すれば5箇所を網羅（再審査で確認）。
- **優先順位**（理論#2）：**カデンツ > 動機保護 > 単一頂点**。カデンツと動機が衝突したら動機を曲げるのでなく §3.1 w0（カデンツ着地k）＋§3.3-2（truncation）で上流解決＝カデンツ句読点を守る。**上流解決が失敗（カデンツpc不到達）した場合、カデンツパス`:1088`の書込は保護ガードを上書きする**（＝li-1が保護音でもカデンツ接近修正を通す・再審査#1）。単一頂点強制は保護音に無効（反復音フックは複数平頂点が正常）。
- 動機**間**の接続音のみ後処理が直す。1拍内の直接対斜（F直後F#等の短調変化形衝突）は接続範囲の掃除対象に含める（理論論点6）。

## 5. U5 展開統合

- varyTail/invert/fragment/sequence は既存mv演算のまま（deg導出ゆえ無改修）。sentence役割（bi/seq/frag/cad `:862-873`）の**動機供給源をpreserve動機へ差替**。
- **exact repetition を役割語彙に追加**（理論#3訂正）：目標は「動機出現 **2.4回/8小節（変形込み**・motif-extraction §4.5の2.4は exact+移高+反行の総数）」であって exact のみを 2.4回にしない。バランス経験則＝**初回反復は exact可・3回目までに何らかの変化**（尾変奏/移高/断片化）＝予測報酬の飽和回避（Huron ITPRA・親doc §2.4）。exact-only分布が要るなら抽出器の展開タグ集計を1本回してから較正（＝実装前の任意前工程）。
- seedMotif（`:793`）／completeMelody（`:1270`）は preserve既定にする価値大（ユーザー動機の同一性保証・§3.4是正）。ただし**preserveを一般生成の既定にするのは耳セッション前は保留**（新経路opt-inのまま）。

## 7. TDD受入条件

1. **bit一致（回帰・最優先）**：motifMode/hook/articulation/inflect 未指定→既存 deep-equal・全既存緑。**hook=0の単一ドロー形をhookロジック追加前に赤で検証**（コード実装順#1）。
2. **動機同度率＋組織化**：hook=1・seed sweepで動機move=0率15-30%帯・**かつ同度の出現位置が句頭/句末に偏る（アンカー性指標）**（理論#4）・連続同度≤3。A塊の反復音位置がA'/A''で同位置再現≥90%（k揃い必須＝ヒステリシス）。
3. **往復再現（本丸合格線）**：seedMotif=オーナー例→提示ブロックの**出力MIDI輪郭が移高を法として逐語一致**（度数でなく実音・3/3 seed＝理論#1）。同degグループが同pitch・輪郭符号不変。

5. **カデンツ着地**：preserve時もフレーズ末が安定度（1̂/3̂/5̂等）に着地。
6. **アーティキュレーション**（U6）：articulation=1で同度連打間 gap≥0.05拍が100%・staccato dur/IOI≈0.4±0.1。既定0でdur完全不変。

## 8. 実装順（コード眼の助言・危険な順に先行検証）
