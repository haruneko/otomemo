# E2E受け入れテスト：一曲書く（今セッション機能の通し・2026-07-13）

**目的**：今セッションで入れた機能を「一曲を頭から書き上げる」実フローで受け入れ確認する。単体テストは全緑だが、
**音色・骨格反復・アルペジオは最終的に耳/目が判定**する層＝実機通しで初めて分かる。ドッグフード＝作りながら壊れを出す。

**運用**：本シナリオを**レビュー**→オーナーが〈曲の素性〉と〈セクション役割〉を指定→**役割を割り当てて E2E 実行**。
各フェーズに「誰が/何で駆動」「合否」欄。❌は原因を書いて Task 化。

## 1. 曲の素性（← オーナーが指定・下は既定の叩き台）
- **調/拍子/テンポ/ムード**：A minor ／ 4/4 ／ 90bpm ／ 切ないバラード ← 差し替え可
- **セクションと役割（順序）**：`Verse(A)` → `Chorus(B)` → `Chorus(B')`（サビ繰り返し）← **役割はオーナー指定**
- **編成（各セクション）**：メロ／ベース／ドラム／**コード楽器1＝ピアノ（GM 0・strum）**／**コード楽器2＝ハープ（GM 46・arp）**
- **ハープの効かせ所**：サビ(Chorus)で **駆け上がり幅3oct・区切り1.5拍**（Verse は控えめ or 無し）← 差し替え可

> 役割割り当て欄（レビュー後に埋める）：
> - Verse の役割＝`verse`／energy＝ ／ハープ＝
> - Chorus の役割＝`chorus`／energy＝ ／ハープ＝幅__oct・区切り__拍
> - （追加セクションがあれば）＝

## 2. 前提（スタック）
- api/web 常駐（systemd cm-api/cm-search）。web は本コミット時点で `pnpm --filter web build` 済。
- **フィール/音色は api 稼働で反映**。コード楽器2音色の修正を試すので、api は最新 dist を配信していること。
- 作業先の器（プロジェクト）を1つ決める（無ければ新規）。以下ネタは全て project スコープ・その器に紐付け。
