# メロディ・コーパス追試（POP909・独立再現）

対話で導出したメロディ生成の知見を、計測スクリプトを**ゼロから書き直して独立に追試**した記録。
数値が一致するか、しない所はどこか、を正直に残す。整列の肝・各計測・統合モデル・再現手順の順。

## ① 方法

### データ
- POP909（先頭250曲をスキャン）。各曲: `NNN.mid`（メロは track 名 "MELODY"）/ `beat_midi.txt`（1行=1拍, col0=秒, col2=downbeatフラグ）/ `chord_midi.txt`（`開始秒 終了秒 ROOT:quality`, `N`=無）/ `key_audio.txt`（先頭行3列目 `ROOT:mode`）。
- メロ抽出: `parseMidi` → `notesOfTrackNamed(parsed,"MELODY")` → `skylineMelody(...)`。note.start は**拍**単位。
- 4/4のみ対象（`beatsPerBarFromBeats===4`）。

### 整列の肝（全実験の前提）
MIDIの拍時計と秒注釈（chord/beat）は**曲ごとに整数拍ズレる**。各曲で `(k∈-8..8, phi∈0..3)` を
**強拍（拍内位置≈0と≈2）のコードトーン率最大**で探索し、**rate≥0.85 の曲だけ採用**。
- phi=拍0基準の位相、コードは `chordAt(note.start + k)`（chord秒→拍は beat_midi 秒列を線形補間）。
- これをやらないと「強拍<弱拍」等の偽結果になる。
- chord pcs は root+quality（maj/min/maj7/min7/7/dim/aug/sus2/sus4/6/hdim7 等を実装）。

**採用結果: 250曲中 71曲を採用**（期待 ~73曲、ほぼ一致）。採用曲の整列時・強拍コードトーン率の平均=**0.922**。
k は曲ごとにバラける（k=0が24曲、k=-1が15曲、他は±8まで広く分布）＝整数拍ズレが実在することの裏付け。
phi は 0/1 にほぼ二分（35曲/36曲）。

### 注意点
- 「最初の最高音」で頂点を採ると tie/skyline で前倒しの偽像が出るため、輪郭は**平均**で見た（③）。
- 度数化は key root 相対に最近接スケール音へスナップ＋オクターブ連続度数。

### 一致しなかった所・要注意（追試の核心）

**③ アーチ**: 平均正規化輪郭は両端 ~0.50–0.52・中央 ~0.55–0.56 で、**確かに中央が高い緩いアーチ**だが、振幅は小さい（±0.03程度）。フレーズ毎の平均輪郭頂点の平均位置は **0.42**（中央付近、期待と一致）。一方「最初の最高音」位置は 0.41 で、今回のデータでは警告された強い前倒し偽像は顕著には出なかった。**結論=アーチは実在するが浅い**。「頂点中央」は平均輪郭で支持される。

**⑥ stay/move 反転**: onset密度（stay 1.38 / move 3.41）は明確に一致。だが**反転率の絶対値は期待（stay11%/move22%）と一致しない**。
- step毎の反転率では stay=69% / move=50%（**符号が逆**）＝stay区間の数少ない表面音は隣接音への往復装飾で反転だらけ、分母も小さい。
- 「窓に反転を含むか」率では stay=18% / move=40%（**順序は期待通り move>stay**、密＝反転多）。
- ＝**定性的順序（move窓のほうが反転を含みやすい）は再現するが、特定パーセンテージは反転メトリックの定義に強く依存**。期待の 11%/22% は今回の定義では再現せず。

**⑦ 着地方向の度数別**: 全体 overshoot/undershoot（77%/61%）は完全一致。「主音は上から降りる」は **67%** で向きは再現（期待81%よりは弱い）。だが **5度=下から74%・オクターブ=ほぼ下から85% は再現しなかった**：
- 5度は今回 **上から63%**（期待と逆）。
- 高位主音/オクターブは 上48%/下48%でほぼ五分。
- over/under では高位主音 under=72% と「下を経由して上の主音へ駆け上がる」傾向は見えるが、ネット方向としては中立。
- ＝**度数別の「下から/上から」の細かい主張は本追試の整列・着地定義では支持されない（flag）**。着地定義（dur≥2 or 直後gap≥1.5）や approach 窓（前2拍）の取り方に敏感と思われる。

**⑪ 16分slot 語彙**: 中央条件（skMove=±1, +0）は 50%=8-9語で期待（5-7）に近いが、跳躍をまとめる skMove=±3 クランプ条件は distinct が膨らみ 50%=24-30語に。**8分への畳み込みは明確に再現**（50%が一桁台へ激減、top cell が保持→[0@0]/[0@0,0@1]、↑3度→[0@0,2@1] 等、期待例と一致）。

**⑫ subdivided 過多**: subdivided が 13%（期待6%）と多め。onset の 16分量子化で隣接オンセットが両 slot に乗ると過剰計上され得る。**構造的主張（割れ方は常に r01、違いは2音目の度数 move のみ、~5 move で90%）は完全に再現**。割合だけ flag。

## ③ 統合モデル（再現された描像）

メロディは**小語彙の再帰3層・BPM非依存**として記述できる（本追試で支持）:

1. **骨格層（~2拍の move）**: 骨格音は平均 ~1.9拍≈2拍 保持（M4）。2拍骨格の動きは 留37%・順次・跳躍が小語彙で、向きは反転しがち（65%）、三つ組はターン/保持中心（M5）。骨格列は**4小節(lag8)・8小節(lag16)で自己反復**（自己相関ピーク 32%/35%、M10）＝反復＋移調反復の構造。
2. **8分 joint cell 層（骨格moveで条件づけ）**: 1拍の中身を `度数move@slot` で符号化し骨格 move で条件づけると、**8分slotで条件あたり 50%=2-4語・80%=数語**に畳まれBPM非依存化（M11）。保持→`[0@0]`/`[0@0,0@1]`、↑3度→`[0@0,2@1]`、↓2度→`[0@0,-1@1]` 等が支配的。
3. **16分細分層（稀）**: 8分窓は plain(1音)/empty が大半で、subdivided(16分2音)は少数。割れ方は**常に r01**（8分→2つの16分）で、**違いは2音目の度数 move のみ・~5 move で90%**（M12）。

全層で**語彙が小さく、強拍はコードトーン（90.8% vs 弱拍54.6%, M2）に縛られ、同音回帰は上下対称で有界（M9）**。フレーズは浅い中央アーチを描く（M3）。整列（曲ごとの整数拍ズレ補正）が全結論の前提。

## 再現用スクリプトの要点（次の再追試者へ）

- **共通ライブラリ `_lib.ts`**:
  - `loadSong(id)`: MIDI/beat/chord/key を読み、skyline メロ抽出、4/4判定、整列探索 `(k∈-8..8, phi∈0..bpb)` で強拍コードトーン率最大化、**rate≥0.85 で採用**。`makeChordAt(song)` で整列済み chordAt（拍→秒は beat_midi の線形補間 b2s、秒→コードは線形探索）。
  - `skeletonByBeat(notes)`: 各整数拍の直近onset音（M4）。
  - `skeleton2beat(notes, phi)`: phi 基準2拍刻みの骨格列（M5-M11 の基盤）。
  - `collectSongs(250)`: 先頭250曲から採用曲を集める（→71曲）。
- **整列が命**。やらないと強弱が逆転する。`b2s`/`s2b` は beat_midi の秒列（行index=拍）を補間。
- **度数化**: key root 相対 pc を最近接スケール音へスナップ＋オクターブ連続度数（M11/M12）。
- **8分集約**: 16分slot 0,1→8分slot0 / 2,3→8分slot1、同一8分slotは先頭のみ採用（M11）。
- **着地定義**: `dur≥2 or 直後gap≥1.5拍`。approach は前2拍窓（M7/M8）。**度数別方向の主張はこの定義に敏感（再現弱い）**。
- **反転率は定義依存**（step毎 vs 窓に含むか）で大きく変わる（M6）。報告時は定義を明記すること。
- 各計測は `cd apps/api && npx tsx _mN.ts` で実行。DB・データは読むだけ。
