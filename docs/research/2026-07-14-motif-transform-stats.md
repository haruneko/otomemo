# POP909 実測: モチーフ展開の文法（変換型の頻度・距離分布・連鎖）(2026-07-14, M9)

**問い**: ポップスの実作で、モチーフ（0.5〜2小節の小単位）はどの変換でどの頻度で再利用されるか。
セクションレベルの反復統計（POP909: 20%完全反復/29%suffix/18%prefix、2026-07-14 skeleton-dictionary-metrics）は既知。今回はその**一段下＝モチーフレベル**の変換文法を全909曲から実測した。

**結論先取り**:
1. ポップのモチーフ変奏の主要通貨は**「音数±1〜2の増減を伴うゆるい変奏」**（装飾音の追加/削除）＝全関連ペアの6〜7割。教科書的変換（反行・拡大・縮小）は**実質ゼロ**（≤0.06%）。
2. **「近くでは変える・遠くでは戻す」**：完全反復(literal)は隣接距離でシェア0.4〜1.4%しかなく、5小節以上離れて（＝セクション回帰で）14〜16%に跳ねる。
3. 移高（ゼクエンツ）は**上行2度が圧倒的**（2小節窓で+1/+2半音＝82%、下行2度は1%台）。
4. 連鎖は**累積ドリフトせず原型回帰**：モチーフ族の80〜92%が後半で原型へ戻る。原型類似は 2回目0.93→**3回目0.95**→4回目以降0.90＝「A A' A」型が規範。

## 1. 手法

- **データ**: POP909 全909曲の MELODY トラック（`curl -sL https://codeload.github.com/music-x-lab/POP909-Dataset/tar.gz/refs/heads/master`、リポジトリ https://github.com/music-x-lab/POP909-Dataset ）。
- **位相**: 根治済みの注釈拍写像を流用（`apps/api/src/music/phrase.ts` の `beatTimesFromBeats`＋`remapToBeatGrid`＋`firstDownbeatFromBeats`。R0・2026-07-14 と同一パイプライン）。MIDI tick→秒→注釈拍座標で downbeat が整数拍に乗る。
- **モチーフ化**: 実 downbeat アンカーの非重複グリッドで 1小節窓と2小節窓の2通りに分割（最少3音/4音、同音連打・ペダル区間は除外）→ `scoreDurations` で楽譜長復元。曲内比較なので移調正規化は不要（同一キー内）。
- **反復対検出**: 曲内の全モチーフ対を総当り。安いゲート（音程ヒストグラム cos ≥0.35）→ ルールカスケードで分類。関連なしは捨てる。類似度は既存 `melodySimilarityLayered`（音程0.5+リズム0.3+輪郭0.2、移調不変）。
- **変換分類**（優先順のカスケード。A=先行、B=後続）:
  | 型 | 判定 |
  |---|---|
  | literal | 音高列・オンセット列とも完全一致 |
  | transpose | オンセット一致・全音符が一定半音シフト |
  | inversion | オンセット一致・音程列が符号反転 |
  | cadence_swap | 頭≥50%完全一致・末尾1〜2音のみ差し替え |
  | rhythm_keep_pitch_var | オンセット一致・音高が変形（関連あり） |
  | pitch_keep_rhythm_var / augmentation / diminution | 音高列一致・リズム変形（オンセット幅比≈2/≈0.5 で拡大/縮小） |
  | fragment_head / fragment_tail | Bが短く、音高列がAの先頭/末尾部分列 |
  | expand_elaborate | Bが長く、Aを頭/尻に含んで延長 |
  | developing_var | 同音数・リズムも音高も少し違う（sim≥0.62） |
  | length_var | 音数が違うゆるい変奏（sim≥0.6、断片でも延長でもない） |
- **連鎖**: sim≥0.8 で連結したモチーフ族（3回以上出現）ごとに、出現順で原型（初出）との類似推移と隣接変換を追跡。
- **統計のみ保存**。リテラルな旋律は一切保存しない。集計JSON:
  - `/tmp/claude-1000/-home-shuraba-p-projects-creative-manager/df6fed86-2c29-4418-8ade-ff12c85794b2/scratchpad/motif-transform-stats-1bar.json`
  - `/tmp/claude-1000/-home-shuraba-p-projects-creative-manager/df6fed86-2c29-4418-8ade-ff12c85794b2/scratchpad/motif-transform-stats-2bar.json`
  - スクリプト: 同ディレクトリ `motif-transform.ts`（`cd apps/api && node_modules/.bin/tsx <script> <1|2>` で再現）

**規模**: 1小節窓＝49,968モチーフ（55/曲）・関連対472,889。2小節窓＝27,768モチーフ（30.5/曲）・関連対191,845。909曲全数。

## 2. 変換型の頻度（関連ペア内シェア %）

| 変換型 | 1小節窓 | 2小節窓 | 読み |
|---|---:|---:|---|
| **length_var**（音数違いのゆるい変奏） | **61.6** | **73.4** | 支配的。§3参照＝大半は±1〜2音 |
| literal（完全反復） | 11.7 | 8.3 | 距離依存が強い（§4） |
| developing_var（同音数・リズムも音高も微変） | 10.0 | 8.9 | |
| rhythm_keep_pitch_var（リズム保存・音高差し替え） | 8.4 | 3.8 | コード追従の再ハーモナイズ型 |
| expand_elaborate（延長・装飾展開） | 2.4 | 2.1 | |
| cadence_swap（終止だけ差し替え） | 1.4 | 0.5 | 少数だが近距離に偏る（§4） |
| fragment_head（頭だけ断片） | 1.3 | 1.2 | |
| transpose（定数移高＝ゼクエンツ） | 1.1 | 0.5 | 上行2度が圧倒的（§5） |
| fragment_tail（尻だけ断片） | 1.1 | 0.6 | |
| pitch_keep_rhythm_var（音高保存・リズム変形） | 0.9 | 0.7 | |
| inversion（反行） | 0.06 | 0 | **実質不使用** |
| diminution / augmentation（縮小/拡大） | 0.01 / 0.01 | 0 / 0 | **実質不使用** |

- **リズム保存・音高変形（8.4%）は音高保存・リズム変形（0.9%）の約10倍**＝ポップではモチーフの同一性を**リズムが担う**。音高はコードに合わせて差し替える。
- 反行・拡大・縮小というクラシックの展開技法はポップの実践にほぼ存在しない。生成メニューに入れると「ポップらしさ」から外れる。

## 3. length_var の中身＝音数ジッタ

length_var の音数差（後続−先行、±6クランプ）:

| |Δ音数| | 1小節窓シェア | 2小節窓シェア |
|---|---:|---:|
| 1 | 50.5% | 32.5% |
| ≤2 | 79.8% | 56.6% |
| ≥5 | 1.9% | 15.3% |

- 1小節窓では**8割が±1〜2音**＝「装飾音を1つ足す/抜く」「シンコペで1音割る/結合する」レベルの微変。増減はほぼ対称（+52.3% / −47.7%）。
- 2小節窓で大きい差（≥5音）が増えるのは、後半1小節を丸ごと変える型（＝period の consequent 差し替え等）が混入するため。
- **含意**: 「音数ジッタ±1〜2」はポップの変奏の一級市民。歌詞の字数対応（メリスマ⇄シラビック）とも整合する。

## 4. 距離別の変換分布——「近くでは変える・遠くでは戻す」

各距離バケツ内での型シェア（%、1小節窓/2小節窓）:

| 型 | 隣接(≤窓幅) | 2-4小節 | 5-8小節 | 9-16小節 | 17-32小節 | >32小節 |
|---|---:|---:|---:|---:|---:|---:|
| literal | **1.4 / 0.4** | 10.2 / 6.4 | **16.1 / 14.1** | 7.4 / 5.3 | 13.4 / 9.6 | 12.4 / 8.7 |
| length_var | 71.3 / 79.7 | 56.5 / 66.5 | 58.7 / 66.0 | 67.3 / 78.5 | 61.5 / 73.1 | 60.6 / 73.9 |
| rhythm_keep_pitch_var | 9.8 / 6.0 | **12.2 / 7.0** | 7.6 / 4.3 | 7.9 / 3.2 | 7.4 / 3.0 | 8.5 / 3.4 |
| cadence_swap | 1.6 / 0.6 | **2.7 / 1.2** | 1.6 / 0.9 | 1.1 / 0.4 | 1.2 / 0.4 | 1.4 / 0.4 |
| transpose | **0.0 / 0.0** | 0.7 / 0.1 | 0.7 / 0.4 | 1.2 / 0.5 | 1.1 / 0.5 | 1.4 / 0.6 |
| developing_var | 10.9 / 10.0 | 10.1 / 10.3 | 9.0 / 7.9 | 10.4 / 8.6 | 10.0 / 9.0 | 10.1 / 8.8 |

（バケツ内ペア数: 1小節窓 13,575〜157,253 / 2小節窓 12,593〜62,853。フルの表はJSON `catByDist` 参照）

- **完全反復は隣接でほぼ起きない**（1.4%/0.4%）。直後にもう一度言うときは必ずどこか変える。完全反復のピークは**5-8小節**（period の前後半対応・8小節単位の回帰）と**17小節以上**（セクション回帰）。
- **cadence_swap と rhythm_keep_pitch_var は2-4小節距離で最頻**＝「2小節言って、2小節後に終止/音高だけ変えて言い直す」period 的な使い方。skeleton-dictionary-metrics のフォーム回帰4変換のうち cadence-swap の実測的裏付け。
- transpose（ゼクエンツ）は隣接でゼロに近く、9小節以上で増える＝隣り合う小節を階段状に上げる古典的ゼクエンツより、**離れた再演で高さを変える**使い方が主。

## 5. 移高（ゼクエンツ）の半音分布——上行2度バイアス

| シフト | 1小節窓 (n=5,208) | 2小節窓 (n=870) |
|---|---:|---:|
| +1 / +2（上行2度系） | 34.1% / 23.1% → **計57.2%** | 45.5% / 36.6% → **計82.1%** |
| −1 / −2（下行2度系） | 0.2% / 1.4% → 計1.7% | 0 / 1.3% |
| ±5・±7（4度/5度） | 各6〜8% | 計5.6% |
| ±12（オクターブ） | 10.9% | 8.4% |

**モチーフを移高するならほぼ「上へ2度」**（サビへ向かう盛り上げ・問いかけの反復上行）。下行ゼクエンツはポップの実作にほぼない。4度/5度はコード進行追従（IV/V 上での再演）、オクターブはレジスタ替え。

## 6. 変換の連鎖——累積ドリフトせず「A A' A」で原型回帰

sim≥0.8 連結のモチーフ族（3回以上出現）を出現順に追跡:

| 指標 | 1小節窓 | 2小節窓 |
|---|---:|---:|
| 分析した族 | 7,096 | 3,386 |
| 後半で原型へほぼ完全回帰（sim≥0.97）する族 | **91.7%** | **80.4%** |
| 純粋ドリフト（戻らない）族 | 8.3% | 19.6% |

原型（初出）との類似の推移（族内の出現位置別平均）:

| 出現位置 | 1小節窓 | 2小節窓 |
|---|---:|---:|
| 1回目（原型） | 1.000 | 1.000 |
| 2回目 | 0.948 | 0.932 |
| **3回目** | **0.963** | **0.952** |
| 4回目以降 | 0.900 | 0.896 |

- **単調に離れていかない**。2回目で変形し、**3回目で原型へ寄り戻す**＝「提示 A → 変奏 A' → 回帰 A」が統計的規範。変形が累積していく（A→A'→A''→…）モデルは実作と合わない。
- 族内の隣接遷移の型: literal が49.0%（1小節窓）/39.1%（2小節窓）で最多、残りは length_var・rhythm_keep_pitch_var 等の微変。**族の中では「そのまま or 微変」だけが起き、大変形は族の外**（＝別モチーフ）になる。

## 7. 生成への含意（モチーフ展開ノブ / skelForm 拡張）

1. **変換メニューの実測重み**（モチーフ再演サンプラの初期値）:
   - 音数ジッタ±1〜2（装飾音の追加/削除）**≈60%**・完全反復≈12%・同音数微変≈10%・リズム保存音高差し替え≈8%・断片/延長≈5%・終止差し替え≈1.5%・移高≈1%。
   - **反行・拡大・縮小は出さない**（ポップの語彙にない）。
2. **距離条件付きサンプリング**: 変換の選択を「前回出現からの距離」で条件付ける。
   - 隣接（直後）: literal 禁止に近い（≤1.4%）。音数ジッタ or リズム保存音高差し替えで必ず崩す。
   - 2-4小節: cadence_swap / rhythm_keep_pitch_var の適所（period の consequent）。
   - 5-8小節・セクション回帰: literal 解禁（14〜16%）＝「戻る所ではちゃんと戻る」。
3. **A A' A 原則**: 同一モチーフを3回以上使うときは、3回目を原型回帰させる（2回目より原型に近く）。変形を累積させるチェーンは統計的少数派（1割）。skelForm の反復スケジューラに「回帰フラグ」を持たせるのが素直。
4. **リズム＝同一性、音高＝可変**: モチーフ変奏を実装するなら「リズム骨格を固定して音高をコードへ再フィット」が最頻の実作手筋（音高保存リズム変形の10倍）。既存の fit_to_chords 系と相性が良い。
5. **ゼクエンツは上行2度のみ**をデフォルトに（+1/+2半音で57〜82%）。下行ゼクエンツ・反行ゼクエンツはノブの奥にしまう。
6. skeleton-dictionary-metrics のフォーム回帰4変換（literal/cadence-swap/fragment-accel/contrast）との整合: literal と cadence-swap は実測で裏付け（cadence_swap は2-4小節距離に偏在＝period 用途）。fragment 系は頭断片1.3% > 尻断片1.1% と僅差で、**頭を残す断片化が優位**（Caplin の fragmentation とも整合）。contrast は「関連なし」側なので本統計の対象外。

## 8. 注意（この統計の限界）

- 分類はルールカスケード＝閾値依存。length_var は「関連あり・音数違い・断片でも延長でもない」の受け皿で、中に多様な変奏を含む（§3の音数差分解で主成分は±1〜2音と判明）。
- ペアレベル集計は大きいモチーフ族を二乗で重く数える（k回出る族は k(k−1)/2 ペア）。族単位に正規化した見方は§6の連鎖分析側。
- 非重複グリッド窓なので、小節境界をまたぐモチーフや0.5小節単位の反復は 1小節窓の解像度未満。skyline メロ抽出・`scoreDurations` 量子化の癖も継承する。
- POP909 は中国語ポップ909曲＝ジャンル偏りあり。J-Rock/アニソン等への外挿は傾向レベルで。
