## 棚卸しの要点（正典＝本doc下部の参照リスト）
- **二段ドロップダウンの前例**：MetaPanel の音色ピッカーは「歌声を選んだ時だけ『声』ドロップダウンが出る」（案B二段・`MetaPanel.tsx:173-200`）＝「音色を選ぶと関連ノブが条件出現」の直接の雛形。
- **ハブ契約**（`TinkerSheet.tsx:8-15`・コード内不変条件）：ハブに足してよいのは新パーツのタイル+1のみ。新ノブは引き出しの中（前面 chip6±1・seg1行まで・超過は群アコーディオンへ）。**奏法を独立タイルにするのは契約違反**。
- **ChordPatternEditor**：響きゾーンは3行（③行はmode分岐で既に満杯）。行を足すならCP版の行契約（最大N行）の新設が要る。タップ標的28px＝詰め込み耐性は低い。
- **試聴の型**：候補カード＋▶（`auditionCandidate`）が「聴いて選ぶ」の既存動線。MiniRoll は表示専用。
- **即時プレビュー**：ChordPatternEditor はノブを触った瞬間 `previewNote` で和音が鳴る＝奏法変更の手応えに流用可。

## 設計原則（コンセプトから）
1. **ゼロ操作でそれっぽく**：出先メモの主動線では、ノブを1つも触らずに「楽器を選んだらその楽器らしい演奏」が既定で付くべき。ノブは家に帰ってからの微調整。
2. **選ぶのは耳**：型辞書（28型）を名前の列挙で選ばせない。**聴いて選ぶ**（候補カード＋▶の型）。
3. **肥大化ガード**：ハブ契約遵守・CPエディタに行契約を新設・トップ画面には一切触れない。

## 案（3つ＝役割が違うので排他でなく組合せ候補）

### 案A：音色→奏法の連動（既定を決める層）
MetaPanel の音色ピッカーに **「奏法」二段ドロップダウン**を追加（声ドロップダウンと同型）。
- 音色ファミリ→奏法既定の写像（Guitar系→`style:guitar`＋ストラム既定／Piano/EP系→`keyboard`＋コンピング既定／Pad系→白玉）＝Yamaha NTR/NTT の「楽器役割ごとに変換規則」の思想そのまま。
- 明示選択もできる（既定＝音色おまかせ）。保存はネタ属性（program と並ぶ）。
```
音色 [ Steel Guitar ▼ ]
奏法 [ おまかせ（ギター・ストローク） ▼ ]   ← 音色がギター系の時だけ出現
```
- 利点：**ゼロ操作の主動線が成立**・UI増加は条件出現1行のみ。欠点：奏法の細部（strumMs等）はここに置けない（置くと肥大）。

### 案B：ChordPatternEditor に「奏法」行（微調整の層）
響きゾーンに第4行として奏法 seg（鍵盤/ギター）＋ギター時のみ「じゃら〜ん」スライダ（strumMs 0/弱/中/強）。
- **同時にCP行契約を新設**：「響きゾーンは最大4行・超過は群アコーディオンへ沈める」を TinkerSheet ハブ契約と同文体でコード内に明記。
- 触った瞬間 previewNote で鳴る（既存挙動を流用）＝スライダの手応えが即わかる。
- 利点：編集画面で完結・即時フィードバック。欠点：モバイル縦伸び（内部スクロールで致命的ではないが密度悪化）。

### 案C：型は「聴いて選ぶ」トレイ（選択肢の層・S2と直結）
TinkerSheet のコード引き出しに「パターン」＝ジャンル/場面 chip（バラード/ロック/シティポップ/4つ打ち…）→ genPart で型辞書から候補3-4件生成→**候補カード＋▶試聴→採用**。
- ベース引き出しの「ジャンルchip6＋型直指定は沈み」の既存構造と同型＝一貫性。型名の直指定 select は沈み（群アコーディオン）へ。
- 利点：28型を名前で覚えなくてよい・「候補まで機械・選ぶのは人間」の思想通り。欠点：S2（型辞書）完成が前提。

### 推奨＝A＋C を本線、B は薄く
- **A が既定（ゼロ操作）・C が選択（聴いて選ぶ）・B は微調整（最小1行だけ）**という三層。どれも既存の型（二段ドロップダウン/候補トレイ/CP行）の流用で新発明なし。
- 実装順の提案：A（小・S1の style を即露出）→ B薄（seg＋スライダ1行＋行契約）→ C（S2完成後）。
- ベース側の空き地（kickLock/snareGap/approach/slashBass のUI未露出）は**ベース引き出しの「細かく」群**へ沈める（前面chip契約を守る）＝独立スライスで随時。

## オーナーに聞きたいこと（帰宅後でOK）
1. 三層構成（A既定/C選択/B微調整）で違和感ないか。「奏法はネタ属性（音色の隣）」という置き場の感覚が合うか。
2. 案Aの「おまかせ」文言と、音色→奏法の既定写像の粒度（ファミリ単位で十分か）。
3. ベース×ドラムノブ（kickLock等）の露出優先度＝今回まとめてやるか・後回しか。
