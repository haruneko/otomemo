**開く**：section を開く→**骨格ブロックをタップ**→全画面の「机」が開く（起動時に自動でループ再生が始まる）。

| # | 操作 | 期待（合格判定） | ○/× |
|---|------|------------------|------|
| 1a | ループ再生中、ロール上で**骨格の打点を1つ足す/動かす** | 打ち終えて**約0.4秒後**、**再生位置が頭に戻らず・音が途切れず**、次の周回から新しい骨格が鳴る | |
| 1b | 立て続けに数点いじる（ドラッグ含む） | 打鍵のたびに鳴り直さず、**手が止まってから**まとめて反映（debounce）。ガタつき・二重発音なし | |

## S2：骨格 S2 較正（注意色＋導出→明示境界の手触り）
机 or 単品 SkeletonEditor で、**メロ骨格点とベース（実効ベース）の対位**を見る。

| # | 操作 | 期待（合格判定） | ○/× |
|---|------|------------------|------|
| 2a | 強拍に**コードの非構成音**が来る骨格点を作る（対位ガター/接点に音程バッジが出る） | **強拍かつ不協和**の点だけ**注意色**が付く（`dissonant = 非協和 && 強拍`）。弱拍/協和には付かない | |
| 2b | 注意色の鳴り具合を耳で確認（接点タップ→その瞬間だけ2声ダイアッド試聴） | 色が付いた点が**実際に濁って聞こえる**か（過検出＝良い音に色／見逃し＝濁ってるのに無色 を探す） | |
| 2c | **明示ベース点**を1つ置く→次の明示点/句末までの区間 | その区間は**明示ベースが支配**、句末で**導出（コード由来）ベースに戻る**。境界で不自然な跳躍がないか | |
| 2d | 明示ベースを消す | 全区間が**導出ベース**に戻る（コード root 代用含む）。対位バッジも追従して更新 | |

**耳/手の的**：注意色の**閾値が妥当**か（強拍不協和の定義が実感と合うか）。導出↔明示の**境界則**（次の明示点 or 句末まで）が手触り良いか＝backlog「境界則=暫定・耳較正で見直し可」の判定。
