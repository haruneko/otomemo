## 1. メロ音価の実測（POP909由来・pop 1087句/23,234音・game 100句は参考別集計）

### 1.1 音価(dur)ヒストグラム（pop）
| dur(拍) | 割合 | | dur(拍) | 割合 |
|---|---|---|---|---|
| 0.25(16分) | 20.1% | | 1.5(付点4分) | 8.1% |
| 0.5(8分) | **52.5%** | | 2(2分) | 0.7% |
| 0.75(付点8分) | 6.2% | | 3(付点2分) | 0.5% |
| 1(4分) | 11.9% | | 4(全) | 0.01% |

- 8分中心（過半）＋16分20%＋**付点系合計14.8%**。長音(≥2拍)はわずか1.2%＝白玉は稀。IOI もほぼ同形（重なり少・単音進行）。
- game は16分31%・付点8.5%・長音4.7%＝**16分寄りで付点少**（別キャラ・混ぜない）。

### 1.2 句内の長短対比（★目標値の中核）
| 指標 | pop 中央値 | p25–p75 |
|---|---|---|
| **変動係数 CV(=std/mean)** | **0.509** | 0.468–0.583 |
| max/min 比 | 4.0 | 3.0–6.0 |
| 音価の種類数 | 4.0 | 3–4 |

- **CV<0.15（ほぼ一様）の句＝0.0%**。実メロは必ず音価がばらける。
- 隣接で dur が変わる率 **53.3%**（同一持続46.7%）＝刻みっぱなしでない。long→short 25%／short→long 28%（対称）。

### 1.3 アゴーギク位置（最長音の位置・ユニーク最長のみ）
- 位置分布: 先頭1/4 20.6%／**中間 45.3%**／末尾1/4 34.1%＝**最長音は中〜後半寄り**。
- **句頭が最長＝わずか3.0%**。句最終音が最長＝27.0%。
- **句末音の長さ ÷ 句平均dur ＝ 中央値 1.45**＝**J-POPは句末を平均の約1.5倍に伸ばす**（game は1.00＝伸ばさない）。

### 1.4 一様生成とのギャップ
一様生成器（全音符同長）＝CV 0。実pop 中央 **CV 0.509**。**ギャップ +0.51**＝「機械的」の正体はCVが0に張り付いてること。

## 2. 和声リズムの実測（在DB chord_progression library・210進行/1494コード・中央5コード）

- **小節あたりコード数(cpb)**: 中央1.75。~1.0=20%／~1.5=27%／**~2.0=53%**。0.5(2小節1和音)は0件。＝密度は「2/小節」寄りで**変化は実在**。
- **コード dur ＝ 事実上 {2,4}の二値**（半小節50%/1小節48%・稀に3）。半小節格子に量子化＝4分以下の細かい交替も1小節超の保持も**無い**。
- **変わり目は100%強拍**（小節頭75%/半小節25%）・弱拍0%。
- **⚠️終盤加速(ii-V圧縮)＝0/210**。むしろ冒頭が速い減速型のみ。**R0再構築で加速が潰れている**（半小節格子＋弱拍0%＋末尾分割0件）＝「加速テンプレ」の目標データは**コーパスに無い**（元拍を保った再取込 or 外部 or P2自作 が要る）。
- 判定: 「全部1/小節に正規化」の疑いは**否定**（1/小節完全正規化は42/210=20%のみ・半小節交替は実在）。だが解像度は半小節止まり。

## 3. 理論（出典つき・強度を明示）

- **① アゴーギク・アクセント**（強い/一次）: 相対的に**長い音が知覚アクセント**（Lerdahl-Jackendoff の phenomenal accent）。実測寄与＝Pitch contour 9.6% > **Timing(dur/IOI) 7.0%** > Tempo 4.5% > Phrasing 4.4% > Meter 3.9%＝**長音は拍の重みより効く**。「短の後の長」r≈0.48。出典 https://pmc.ncbi.nlm.nih.gov/articles/PMC6566653/ 。**含意**＝長音価は構造的に重要な音（句末着地・和声安定音）に置く＝durが構造を補強。
- **② 音価対比＝歌の呼吸**（実務・二次）: 旋律の面白さは長短の**対比**。句レベルの反復輪郭＝**長アンカー→内部の細かい動き→句末の長い着地**。休符は能動的な間。出典 https://www.secretsofsongwriting.com/2010/02/25/rhythmic-contrast-the-long-and-the-short-of-it/ 。**含意**＝リズムを「一律グリッド」でなく**句ごとの durational envelope**でモデル化。
- **③ GTTM Grouping Preference Rules（強い/一次寄り）**: **GPR2b(IOI)＝長い音の後にグループ境界が知覚される**／GPR3d＝音価の変化が境界を作る。出典 https://escholarship.org/content/qt7nn5m93s/qt7nn5m93s.pdf 。**含意**＝意図した句境界に長音/休符を置くと聴者の句切れと一致。逆に**句中に孤立した長音を置くと偽の境界**が聞こえる＝長音は境界に寄せる。
- **④ Huron（弱・方向性のみ）**: 時間的期待＝**「規範を作ってから逸脱」**が表情を生む。均一でも純ランダムでもなく「セクションの音価規範→意図的逸脱」。出典 https://mitpress.mit.edu/9780262083454/sweet-anticipation/ 。
- **⑤ 和声リズム（一貫して attested）**: 速い=推進/動き・遅い=静的（**相対**知覚）。**終止に向けた加速(harmonic acceleration)**。変わり目は**強拍/小節頭**が原則（弱拍=harmonic syncopation は避ける）。テンポと逆相関（速テンポ=遅い和声リズム）。出典 https://musicstudent101.com/121-harmonic-rhythm.html ・https://en.wikipedia.org/wiki/Harmonic_rhythm 。**含意**＝和声リズムを可変軸に。強拍既定＋**終止手前の加速カーブ**＋セクションエネルギー連動＋**メロの長音アンカーを和声リズムの遅い所に、細かい所を速い所に**噛み合わせる。
- **⑥ J-POP実務（和声は堅い/メロ音価は出典薄）**: 王道進行・トニック回避で前進感。**メロの音価対比/小節別和声リズムの定量は英語圏の出典が薄い→理論の一般則＋実測コーパスで埋めるべき**（本doc §1が正にそれ）。

## 4. データ×理論のクロス検証（★重要＝両者が一致）

| 実測(§1-2) | 理論(§3) | 一致 |
|---|---|---|
| 句末音1.45倍（句末を伸ばす） | ①長音=アクセントを着地に／②長い着地／③GPR2b 境界は長音の後 | **強く一致** |
| CV0.51・一様句0% | ②音価対比は必須 | **一致** |
| 最長音は中〜後半・句頭最長3% | ②の「句頭アンカー」は**弱い**＝**J-POPは着地>頭**（データが理論を J-POP 向けに補正） | データ優先 |
| 和声変わり目100%強拍 | ⑤強拍原則 | **一致** |
| 終盤加速0%（コーパス欠落） | ⑤加速は理論で強く支持 | **理論◯・素材× → 決定的テンプレで補う** |

**結論の芯**：メロ音価は「句末を伸ばす＋句内をばらけさせる（CV0.5）」が**実測でも理論でも堅い**。和声リズムは「密度差＋強拍」は裏取り済み、「加速」は理論支持だがコーパス欠落＝**決定的テンプレ or P2自作コーパス**で入れる。

## 5. 設計案

### 5.1 メロ音価＝`rhythmicContrast` ノブ（既定0でbit一致・優先実装）
- **目標**（§1）: 句内 **CV 0.47–0.58**・音価種類 **3–4/句**・隣接dur変化率 **~50%**・**句末音 ≒ 平均の1.5倍**。句頭アンカーは必須にしない（データ3%）。
- **効かせ方**（音価は導出=gap-fill ゆえ onset間隔で作る）:
  1. **句末着地の延長**（最優先・最も堅い）＝句末onsetの後を空けて最終音を伸ばす（既存 `flow` の強化/一般化）。
  2. **句内の粗密**＝内部に細かい動き（16分/8分）を残しつつ、句内に1–2個の長め音を確率的に立てる（density と別軸＝総量でなく**分散**を上げる）。GTTM③に従い**長音は境界寄り**（孤立長音を句中に置かない）。
  3. **付点の導入**（実測14.8%）＝long-short対（付点8分+16分等）を語彙に足す。
- **正典の着地点**（実コード）: `melodyCells.ts` の `sampleBarRhythm`（onset語彙）＋ render の dur ループ。**既定0＝現行の gap-fill そのまま＝bit一致**。ノブONで onset間隔に粗密（句末拡張＋句内分散）を注入。TDD＝OFF=bit一致／ON=生成メロの句内CVが 0.4–0.6 帯へ入る（多seed集計）・句末音が平均超。
- **コーパス連携**: 今回焼いた `motif-model.json` の rhythm パターン＝実onset型＝ONの語彙/確率のソースに使える（統計＝コンプラ）。

### 5.2 和声リズム⑨＝可変軸（design案件・大きめ）
- **密度**（裏取り済 §2）: 1/小節↔2/小節（**半小節グリッド**・変わり目は**強拍固定**）。セクションエネルギー連動（静=遅/盛=速）。
- **加速テンプレ**（コーパス欠落ゆえ決定的に）: 終止手前小節を分割（例 最終小節を IV→V or ii→V の2和音に）＝理論⑤の harmonic acceleration を**ルールで**挿す（頻度データが無いのでコーパス確率でなくテンプレ）。⑩(終止前=D)の**リズム版**。
- **メロと噛み合わせ**: メロの長音アンカー（§5.1）を和声リズムの**遅い所**に、細かい所を**速い所**に置く（理論⑤）。
- **実装コスト**: `genChords` が「1小節=1スロット」前提（`generate.ts:348`・funcs/degrees も小節index）＝**度数ウォークをコード枠(slot)単位に作り直す**必要＝既定1/小節でbit一致は保てるが本体は invasive。**メロ音価(5.1)を先に、⑨は後**。

## 6. ギャップ・保留
- **加速の実測目標が無い**（R0で潰れた）＝ii-V圧縮の頻度/位置は**元拍を保った再取込 or P2自作コーパス**でしか取れない。当面は決定的テンプレ。
- **J-POPメロ音価の genre 特定文献が薄い**＝本doc §1の実測（自前コーパス統計）で代替。P2自作コーパスが貯まれば「自分らしい音価」も測れる（伸びしろ）。
- game 群は別分布（16分寄り・句末伸ばさない）＝style で分ける。
