## 0. 要約

2. **骨格は「動かない」が最頻**：同度数保持 bigram が major 25.4% / minor 26.7%。一様ランダムウォークとは全く違う遷移形。開始・終止・遷移の分布は §2 の表がそのまま**生成重みの初期値**になる。

## 1. 方法（再現手順）

- データ: `music-x-lab/POP909-Dataset`（master tarball を scratchpad へ取得）。909 曲、各曲 `NNN.mid`（MELODY/BRIDGE/PIANO トラック）＋ `beat_midi.txt`（1列=秒・3列=downbeat フラグ）＋ `chord_midi.txt`（start秒 end秒 ラベル）＋ `key_audio.txt`（start end Root:mode）。
- 拍写像: MIDI ノート → tick→秒（`makeTickToSeconds` テンポマップ）→ beat 秒列へ内挿（`makeSecondsToBeat`）＝**注釈拍座標**。downbeat＝`firstDownbeatFromBeats` の行 index（この座標では整数拍）。すべて位相バグ根治済（92a4181）の実装を流用。
- メロディ: `notesOfTrackNamed(parsed,"MELODY")` → `skylineMelody`。16 音未満の曲は捨てる。
- 骨格抽出（M1 §5.1 準拠）: anchor から **2拍刻みのスロット頭 H** ごとに 1 音サンプル。
  - H の近傍（−0.3〜+0.5 拍）のオンセット最寄り（tie は長い音）→ `onset`
  - 無ければ H で鳴り続けている音 → `carry`（carry-forward）
  - どちらも無ければ `rest`（欠損として計上）
- フレーズ窓: **8 スロット＝4 小節**、anchor から 4 小節境界に整列。非 rest が半分未満の窓（間奏・空白）は捨てる。
- 度数化: `key_audio.txt` 先頭行（支配的キー）の主音 pc を引いて **key 相対 pc（0..11）**。mode で major/minor 層別（#10）。
- 拍子: 4/4 のみ骨格集計（880/909）。6/4 判定 29 曲は件数のみ。
- 装飾（#11/#12）: 窓内の表面音のうち骨格 onset 以外を装飾とみなし、進入・離脱の音程（順次≦2半音/跳躍≥3）と拍位置（anchor 相対位相の整数偶数拍=強）で分類。

## 3. 設計含意——骨格辞書の中身（数値で確定できるもの）

1. **一様乱数ウォークの置換材料が揃った**: 開始分布（#1）→ `bigramFull` 重み付きウォーク（#3）→ 終止分布（#2）拘束、のパイプラインに入れる数値は JSON にすべてある。**「保持」25% を明示的に持つ**（現在の生成が保持をほぼ出さないなら分布が合わない）。
2. **輪郭 reject sampling の重み**: arch .38 / valley .25 / desc .15 / asc .13 / wave .06 / flat .02（mode 差は小）。
3. **傾性音ガード**: 骨格の開始・終止に 4・7 を置く確率は各 ≤5%。骨格生成で 4/7 着地を強く減点してよい（表面の倚音でなら使う）。
4. **和声整合は「65〜70% コードトーン」が実態**: 骨格を全部コードトーンにスナップするのは硬すぎ。**強拍でも 2 割は非和声（倚音）**＝接点ダイアッド設計に「強拍テンション許容」の余地を数値で与えられる。
5. **レンジ拘束**: 4 小節骨格のレンジは 6〜12 半音に 7 割。14 半音超を出したら reject が妥当。
6. **表面展開の密度ノブ**: 骨格 1 音 → 表面 2.25〜3.5 音（p25〜p75）。展開規則の重みは #12 の表（passing:weak .09 / neighbor:weak .15 / leap:weak .30 …）。**「コードトーン・スキップ」を展開規則に追加**（M1 §4 に無い最大の欠け）。
7. **period リテラル複写の再考**: 実曲の骨格 literal 反復は約 1%。リテラル複写型は「歌モノらしい反復感」の演出としては有効でも、コーパス準拠を名乗るなら **varied（半分一致・12%）相当の「揺らし複写」**が実態に近い。

## 4. M1 仕様の検証——実測してみて見つかった穴（正直な記録）

1. **#2「終止度数」はフレーズ境界注釈なしでは測れない**。本計測は「4 小節窓の末尾」で代用したが、真の句末と一致しない（major で 2 が 18.4% と高いのは半終止と**窓ズレの混合**で分離不能）。POP909 本体にフレーズ注釈は無い。**修正案**: 別リポジトリ `music-x-lab/POP909 hierarchical-structure-analysis` の人手フレーズ注釈を使う、または長音+後続休符で句末を推定する規則を仕様に明記。
2. **#8「反復変換型」の判定仕様が甘かった**。(a) 8 スロット全部が非 rest でないと判定不能＝ **39〜56% が sparse に落ちた**。(b) mod-12 度数で比べるとオクターブ違いの移高が検出できない（transposed 0.2%は過小）。(c) 完全一致要求で literal が過小。**修正案**: rest を許容するアラインメント（類似度 ≥0.8 → literal 級）＋ 2 小節 basic idea 単位（4 スロット）でも判定＋オクターブ付き度数で移高判定。
3. **#9「反復単位長」は未計測**。モチーフ発見（自己相似行列など）が要り、固定窓では数えられない。仕様に「計測アルゴリズム」自体の定義が必要。
4. **carry-forward / rest の扱いは仕様通りで正解**（17.8% / 8.6% と大きい）。ただし「窓の非 rest 率 ≥50%」のような**窓採用ゲート**が仕様に無かった（無いと間奏の空白窓が統計を汚す）。仕様に追加すべき。
5. **転調曲の扱いが仕様に無い**。909 曲中 152 曲（16.7%）は key 注釈が複数行＝曲中転調。本計測は先頭（支配的）キーのみ使用＝転調後の区間の度数が誤る（全骨格音の数%程度の汚染と推定、クロマ度数 2%はここ由来の可能性）。**修正案**: key セグメントごとに度数化するか、転調曲は除外フラグ。
6. **#12 の NCT 分類は旋律文脈だけでは不足**。leap+other が 65% を占め、古典 NCT で説明できない＝**和音照合（コードトーンか否か）を分類に組み込む**べき（コードトーン・スキップと真の逸音を区別できていない）。M1 §4 の表に「chordal skip」行を追加提案。
7. （軽微）#6 の重心は声域の二峰性が出る＝**絶対音高でなく「度数＋オクターブ帯」**で仕様化すべき（M1 は「半音 or 度数」と曖昧だった）。
