# 骨格IF再設計「骨格の机」— 意図メモ（2026-07-12・Fable設計思考）

動くモック＝ [`2026-07-12-skeleton-if-redesign-mock.html`](2026-07-12-skeleton-if-redesign-mock.html)（自己完結・スマホ幅・WebAudioで実際に鳴る）。
目的＝**メロ確定前に、骨格の線が伴奏スケッチに対して対位法的にどう響くかを、①畳み（音程が読める）②実音（編成に乗る）の両方で確かめられる器**。現 `SkeletonEditor`（単品編集・伴奏文脈から切断）の置き換え構想。コードは書いていない（設計思考タスク）。

---

## 0. 結論の一言

**「4つの画面」ではなく「1つの机の4つの前景」**。セクション＝ループするベッド（伴奏スケッチ）を常に足元に敷き、
①ビート ②コード ③骨格 ④表面 は同じタイムライン上の**フォーカス切替**にする。骨格ステージはその机の上の
「対位法の作業面」＝編集面は常に畳み表示・耳は**聴きレンズ**（畳み／実音）で即切替。

---

## 1. pop workflow 突き合わせ（お願い1への所見）

**リズム→コード→骨格→表面の順序自体は実制作と整合**。これは商業ポップスの「トラック先行→トップライン後乗せ」
（track-and-topline）と同型で、理論的裏付け（パルティメント＝外声先行、Ursatz＝2声骨格）も #20 で済んでいる。
順序を変える提案はしない。ただし3点、**捉え方**を直したい：

1. **段はゲートではなく前景**。実作業は「①完了→②へ」と進まない。ループを回しながら
   ②のコードを差し替え→③の響きが変わる→④で吹いてみて→③に戻る、を数分単位で循環する。
   だから器は「ステージごとの画面遷移」ではなく「**1ループ×前景切替**」であるべき。
   在庫不変（realized_from・骨格は残る）という既存設計は、まさにこの循環を支える土台＝活きる。
2. **①リズムは骨格段階では"時間の器"で足りる**。トップラインを書く時に要るのはテンポとノリの
   プレースホルダであって完成ドラムではない。→ ①が空でも③は成立させる（ベッド最小形＝クリック＋コード）。
   段を「空でも進める・薄く埋まる」ようにする＝縦スライス思想と同じ。
3. **②コードは「対位の相手を書く段」だと明示する**。コード先行ポップスでは下声≒コードroot/分数。
   つまり②で分数コード（G/B等）を選ぶことは**下声の対位を書いている**。②の画面に導出ベースを常時表示し、
   「この段が③の相手を決めている」ことを見せる（モック②参照。G↔G/B を切り替えると③のバッジが変わる）。

補：**コードと骨格の先後を固定しない**（#20決定・2声先行→harmonize）は維持。机はコード空でも開く。

## 2. 器の全体像＝「骨格の机」（お願い2）

```
┌──────────────────────────────┐
│ ヘッダ：セクション名・key/meter/tempo         │
│ ステージレール：①ビート ②コード ③骨格 ④表面   │ ←前景切替（画面遷移ではない）
│ ┌─ 前景パネル（③のとき） ─────────────┐ │
│ │ 素材チップ(#s12・他N箇所) ・ 🎲叩き台        │ │
│ │ ツール列：スナップ[2拍|1拍|自由]・声部[メロ|ベース]・[打点|消す] │
│ │ ┌ 横スクロールの机 ─────────────┐  │ │
│ │ │ 句ルーラー（境界・終止ラベル）           │  │ │
│ │ │ 畳みロール：■打点＋▓支配帯・休符ハッチ    │  │ │
│ │ │   実効ベース線（点線=導出/実線=明示）      │  │ │
│ │ │   音程バッジ＋並行/交差の指摘             │  │ │
│ │ │ 休ストリップ                            │  │ │
│ │ │ 接点ストリップ（対位の接点ドット）         │  │ │
│ │ │ ベッド文脈行（コード記号・タップで②へ）    │  │ │
│ │ └──────────────────────┘  │ │
│ │ 実音オーバービュー（常設・非編集・低域ゾーン） │ │
│ └────────────────────────┘ │
│ トランスポート（固定下端）：▶ループ｜レンズ[畳み|実音]｜位置 │
└──────────────────────────────┘
```

### 中心のアイデア3つ

**A. ベッド（伴奏スケッチ）を常に足元に敷く。** 骨格は単品でなく「セクションの中で」書く。
ベッド＝そのセクションの実データ（ドラム・コード・導出/明示ベース・あればコード楽器）。
①②の編集結果がそのまま③のベッドに流れる＝4段が1つの体験として繋がる。ループ再生しながら打点する
（トップライン書きの実作法）。**保存は従来どおり独立素材ネタ**（§4）。

**B. 聴きレンズ＝「ベッドの還元」として定義する（本タスクの設計の的）。** §3に詳述。

**C. 対位法は「接点」として見せ、指摘→即・耳に繋ぐ。** バッジは点でなく**メロ×実効ベースの接点**の属性。
接点ドット（ロール下の専用ストリップ）をタップ→説明ポップ（「強拍の2度。掛留として解決するなら味」等、
**指摘のみ・禁止しない**）→「**この瞬間だけ聴く**」＝当該拍の2声だけを畳みで鳴らすダイアッド試聴。
「どこが引っかかるか（目）→本当に引っかかるか（耳）」を1タップで往復させる。

## 3. 対位法確認 × 試聴2モードの解き方（核心）

### レンズの定義＝「骨格をどう鳴らすか」ではなく「ベッドをどこまで還元するか」

| | **畳みレンズ（音程を聴く）** | **実音レンズ（座りを聴く）** |
|---|---|---|
| 骨格線 | 実音・持続（弦風/GM48踏襲） | 同じ（メロは畳まない＝方式C踏襲） |
| ベース | **実効ベース線を+1octに畳んで持続**（チェロ風/GM42）＝隣接声域で音程が読める・スマホSPの低域問題も回避 | 実域・トラックのノリ（未実体化なら8分パルスで代用） |
| コード楽器 | **ミュート**（3度堆積が音程判別を潰すため） | 鳴らす（かぶり・マスキングを聴く） |
| ドラム | **クリックに還元**（4分・小節頭アクセント＝時間参照だけ残す） | 選択パターンそのまま |
| 聴くもの | 種対位法的な2声の進行（協和/不協和・並行・交差） | 音域の座り・low interval limit・編成での埋もれ |

ポイント：
- **畳み＝2声への還元＋クリック**。従来の「骨格単品の畳み再生」との違いは、時間参照（クリック）と
  実効ベース（明示＋コード導出のマージ）が**セクションの実文脈**から来ること。単品でなく文脈の中の畳み。
- **切替は再生を止めない**。スケジューラは状態を都度読む＝同じループの同じ小節を、畳み→実音→畳みと
  聴き比べられる（モックで動作）。ここが「両方で聴きたい」の実際の使い勝手を決める。トランスポート内の
  大きな2択（親指圏）にした。
- **表示は畳みで固定・実は常設オーバービューに見せる**。編集面がレンズで組み変わると打点の手が狂うので、
  ロールは常に畳み（ベース+2oct表示・計算は実音＝S2決定踏襲）。代わりに**実音オーバービュー**
  （非編集・全声部の実レジスタ＋低域注意ゾーン＋コード楽器の帯）を下に常設し、
  「見た目≠実の高さ」の不安をここで受ける。畳み表示(+2oct)と畳み再生(+1oct)が違うのも凡例で明示。
- 段階分担は #20 踏襲：低域の濁り（LIL）は畳みでは聞こえない→実音レンズ＋オーバービューの担当。

### 対位法チェックの見せ方
- 打点ごとの**音程バッジ**（mod-12単音程・協和=緑/強拍不協和=琥珀/並行・交差=赤・並行は2点を線で結ぶ）。
- **接点ストリップ**＝バッジの要約列。ロールが縦に込み入っても接点だけを1行で俯瞰できる。タップ→説明＋ダイアッド試聴。
- 本格レポート（`analyzeVoiceLeading`）は④の候補カードに添付済（S3d）＝③は軽量な常時フィードバック、
  ④は生成物への総括、と役割を分ける。

## 4. 独立素材との両立（配置越し編集）

- 机での編集対象は「このセクションに置かれた骨格ネタ」。**ビューは配置移調を適用、保存時に外す**
  （melodyPlacementShift 流儀の往復）＝素材は独立のまま、体験は文脈内。
- 素材チップに**「他N箇所で使用」**を常時表示。タップで説明＋「複製して切り離す」（copy_neta→この配置だけ差し替え）。
  使い回し素材を机で直すと他所も変わる、という唯一の罠を明示する（migration時の要実装点）。
- 骨格未配置のセクションで③を開いたら「新規骨格を書く／既存を置く／🎲叩き台」の空状態。
  単品編集（現SkeletonEditor相当）は**素材庫のクイック確認**に降格して残してよいが、主線はこの机。

## 5. 4段の通しの繋がり（モックで触れる形にしたもの）

- ①で選んだビートが②③④のベッドで鳴る（骨格段はクリック代用可の注記つき）。
- ②のコード差し替え（タップ巡回・分数コード含む）が③の導出ベース・音程バッジ・オーバービューに即反映。
- ③🎲叩き台＝候補をループ再生の中で**試着**（現骨格はゴーストで残る）→採用。「句2だけ引き直す」の口も見せた
  （句単位再抽選＝phrases構造の使い道）。
- ④「この骨格で吹く」→候補3つ（対位バッジつき）→**ベッドの上で試聴**（レンズも効く）→置く＝
  新メロ＋realized_from、骨格は不変。④からワンタップで③に戻れる（循環）。

## 6. 状態遷移・操作一覧（移行担当Fable用）

状態：`stage(1-4)` / `lens(fold|real)` / `playing` / `snap(2|1|自由)` / `voice(mel|bass)` / `tool(draw|erase)`
/ `candOpen+candSel+candPreview`（叩き台試着） / `trayOpen+surfPreview+placed`（表面化）。

| 操作 | 遷移・効果 |
|---|---|
| ステージレール タップ | stage切替のみ。**再生・ループ位置は維持**（前景切替の要） |
| ▶ループ / ■ | セクションループ再生。全ステージ共通（ベッド＋現在の骨格/試聴対象） |
| レンズ切替 | 再生を止めずスケジュール源だけ切替（スケジューラは状態を都度読む） |
| ロール タップ（打点） | armed声部にブレークポイントupsert（同点再タップ=削除・同拍別音=移動）。ベースは表示-24でunfold保存 |
| 休ストリップ | armed声部に pitch:null をトグル |
| 句ルーラー 終止ラベル | 半終止⇄全終止（本実装では cadence フィールド） |
| 接点ドット | ポップ（説明・指摘のみ）＋「この瞬間だけ聴く」（2声ダイアッド・畳み） |
| 🎲叩き台→案タップ | candPreview=試着（再生中なら次のスケジュール分から鳴る・現骨格ゴースト表示）。編集開始で自動解除 |
| 採用 | tones置換（在庫はUndo/履歴で戻す想定＝破壊上書きしない流儀） |
| ④吹く→試聴→置く | gen_melody(skeletonNetaId)→候補トレイ（voiceLeadingサマリ添付）→ベッド上で試聴→新メロneta＋realized_from |
| 素材チップ | 使い回し説明＋複製して切り離す |

既存資産との対応（何がそのまま使えるか）：
- 純ロジック＝`apps/web/src/skeletonEdit.ts` ほぼ全部（dominion/explicitBassSegments/effectiveBassAt/
  analyzeCounterpoint/fold/TAP_SLOP/skeletonEarNotes）。モックの機能はこの縮約移植で全部書けた＝**土台は既に在る**。
- 畳み再生＝`skeletonPlaybackNotes(counterpoint:true)`＝畳みレンズの2声部分そのもの。足すのは
  「クリック」と「実音レンズ＝セクション合成再生に骨格線をミックス」（skeletonEarNotes が既にその形）。
- 生成＝gen_skeleton（叩き台）・gen_melody(skeletonNetaId)＋voiceLeadingReport（S3d）・harmonize（コード無し時）。
- 置き換え対象＝SkeletonEditor の「単品の器」という位置づけ（部品は流用可）。SectionEditor 骨格レーンは
  この机への**入口**（骨格ブロックタップ→机が開く）に再定義。

## 7. 趣旨からのズレ提案・気づき（お願い3）

1. **④は「段」というより「③の出口」**。表面化は骨格の机の右端の行為（吹く→試着→置く）であって、
   独立した編集段になるのは置いた後（ノート編集・歌詞乗せ）。モックでは段として残したが、
   本実装では「③の机の中の表面化トレイ」＋「置いた後は既存のメロ編集へ合流」が素直かもしれない。
2. **叩き台の試着（candPreview）を再生を止めずに回す**のは骨格候補・表面候補の両方で効く。
   既存の候補トレイ（reshape-bar型）は静的提示＝「ループ内試着」への格上げが体験を一段変える。
3. **②に導出ベースを露出**（上述）。ついでに「コードをタップ→代替候補巡回」は substitute_chord の
   自然なUI口になる。
4. **接点ダイアッド試聴**は生成候補のレポート（S3d）にも転用できる：spots の各違反に「この瞬間を聴く」。
5. ドラフト規律：机での骨格編集はブレークポイントのままの直接編集（S2決定踏襲）。ドラッグ移動・複数選択は
   本実装での追加（モックはタップのみ）。
6. **8小節等の長尺**：モックは4小節固定。本実装は横スクロール＋ループ範囲指定（「この2小節だけ回す」）が
   欲しい——トップライン書きは短い窓を回して書くため。ループブレース（範囲選択）をルーラーに置くのが素直。
7. 3/4・6/4 は J2a で生成側が受けたので、机のグリッド（snap既定2拍→付点系の調整）だけ拍子対応が要る。

## 8. モックの割り切り（本物との差）

- 音源＝簡易シンセ（本実装はGM音色：メロ48/ベース42踏襲）。ドラム/パッドもプレースホルダ。
- 叩き台・表面候補はベイク済み固定データ（本実装= gen_skeleton / gen_melody(skeletonNetaId)）。
- 4小節・4/4・C major固定。句境界ドラッグ・点のドラッグ移動・複数選択・Undoは未実装（設計は§6に記述）。
- 「置く」後の実配線（capture→place_child→link realized_from）はトーストで示唆のみ。
- タップとパンの区別（TAP_SLOP）は本実装で必須（モックはclickのみ）。
