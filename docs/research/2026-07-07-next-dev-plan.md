## 0. 一行結論

**次のアークは「生成器を増やす」ではなく「足場を組む」。** 監査Aの構造的所見＝**"候補を出す生成は厚い・足場（可視化／評価フィードバック／書きやすさ／自分らしさの燃料）が薄い"**。監査Bの哲学＝**機械は候補/足場/ばらつきまで・仕上げは人間**。両者は完全に同じ方向を指す＝**足場を厚くする**のが次期の背骨。メロ最優先を貫く。

## 1. 監査A（機能ギャップ）要点

## 2. 監査B（あるべき姿）要点

**キーストーン＝「自作 mp3 バックカタログの資産化」。** コンセプト（`requirements.md` L16-20）の努力目標「自分の書いたものに近い音」の**燃料（＝自分らしさ）がまだ空タンク**（過去作はほぼ mp3・`requirements.md` L235）。連想元は今は他人統計（POP909等）。給油装置がちょうど揃った＝**アナリーゼ v1**（mp3→調/コード/輪郭の派生事実）＋**study フレーム（作曲者単位で手癖抽出・#S11改）**。∴ **自作 mp3 → アナリーゼ → study → 個人コーパス/手癖プロファイル** の導線が今初めて開通。他の全テーマは個人燃料が入ると効きが跳ねる。

**テーマ（Bの構想）と統合上の対応：**
- **T1 自作の資産化（キーストーン）** ＝ Aの#22（mp3整理/コード検出 未）を"燃料装置"として一段上に。
- **T2 メロ層を操作面に**（骨格固定・表面N案／変奏ファミリー） ＝ Aの#7/#5/#24。
- **T3 自己進化ループを常設**（over-generate→項目別critique修復→個人らしさで並べ替え→多様性top-k・**総合スコアで1本に潰さない**） ＝ **Aの最痛点#1（評価未接続）そのもの**。部品は完成済、配線が主。
- **T4 足場（scaffold）**（study→生成プリセット／曲全体の構成・転調・対旋律） ＝ Aの#4/#5/#6＋「次の一手」の格上げ。
- **T5 歌詞↔メロを韻律まで**（アクセント輪郭→メロ制約） ＝ Aの#21近傍・安い並行。

**Bの物差し**：`self-check-log` の決定的学び＝**E-rule総合点 ⊥ 耳**（totalでランクするとgaming）／**項目別critiqueは"どの規則が弱いか"を正確に指す**（修復の的）／**外部モデルは専用指標へ降格**。∴「totalで選ぶな・critiqueで直せ・らしさで並べろ・spreadで出せ」。

## 3. 収束点（AとBが独立に同じ結論を指した所）

1. **自己進化ループの配線が最高レバレッジ**：Bの本命テーマT3＝Aの最痛点#1。**評価器は完成済・繋ぐだけ**で「変なメロ検出ガード＋候補の質floor」が全 generate/fit/reshape に効く。コスパ最大。
2. **メロ足場が薄い**：Bの哲学「足場まで」×Aの「足場が空白」。特に**#8 スケール音ハイライト**（純UI・低リスク・"外し音を避ける"直撃）と**#7 非対称フレーズ**（ユーザーmust）は小コスト大効果。
3. **燃料＝自作コーパス**：Bのキーストーン。ループ(T3)の"並べ替え軸"を汎用→本人に化けさせる。単独では大物だが、薄い縦スライス（数曲→個人プロファイル→検索を1箇所バイアス）で燃料の実在をまず作れる。

## 4. 段階計画（メロ最優先・哲学厳守・"部品が既にある"レバレッジ順）

### Phase 0 — 足場の即効ウィン（低リスク・小コスト・哲学ど真ん中）
- **[P0-a] スケール/コードトーン ハイライト**（A#8）＝PianoRoll に調内/コード内の色分け。既存 harmony 情報を描画に足すだけ＝**生成非依存・回帰リスク最小**。「外し音を避ける」書きやすさの土台。
- **[P0-b] 対称⇔非対称フレーズ選択**（A#7・ユーザーmust）＝`skeleton.ts`/`melodyCells.ts` に句長配列を通す（2+2+2+2 固定→4+3等も選べる）。
- **[P0-c] 評価器を MCP `analyze` に露出＋生成の候補選別に1本接続**（A#1・T3の最小スライス）＝`evalMelody`/`corpusTypicality` を「変なメロ検出ガード」として繋ぐ。**作り済みを繋ぐだけ**。
- **[P0-d] doc是正**（A乖離・コード0）＝worker撤去実態・楽譜/systemd未導入を backlog/architecture/status に反映。

### Phase 1 — 自己進化ループを常設（B-T3・P0-c を本配線に）
- over-generate(N) → **項目別critique で決定的後処理修復**（禁則跳躍/gap-fill/強拍CT/単一頂点＝`self-check-log` R-v2-eval で +0.09 実証）→ **E-corpus で"らしさ"順ランク** → **多様性を保った top-k を提示**。`generate/fit/reshape` の裏に固定。**総合スコアは出さない・1本に潰さない**（哲学＆gaming回避）。

### Phase 2 — キーストーン：自作 mp3 の資産化（B-T1）
- 薄い縦スライス：自作数曲 → `analyze_audio` → study（作曲者＝本人）→ **個人プロファイル（コアループ度数shape＋音域＋リズム傾向）を1ネタ保存** → 検索/ランクを"自作寄せ"に1箇所バイアス。**メロは統計/傾向のみ・信頼度ゲート**（MIRのメロ採譜は最弱）。project/library分離厳守（自作燃料と他人統計を混ぜない）。→ **P1のランク軸が汎用→本人に化ける**。

### Phase 3 — メロ層を操作面に（B-T2／A#5・#24）
- 骨格固定・表面N案（`reshape` に layer=surface）／変奏ファミリー（`genFromEssence` 格上げ）／**対照検索**（A#2＝`findSimilar` 反転）。floor(P1)＋個人(P2)が下にあって初めて"使える候補"。

### Phase 4+ — 大物・全体足場
- 曲全体足場（構成/転調/対旋律・A#4/#6・study→styleプリセット）／仮歌＋WAV書出（A#17/#13＝土台共有でセット）／版管理undo（A#12）／情報収集のパーソナライズ（A#21）。

**通奏低音（安い並行）**：**T5 歌詞韻律**（アクセント輪郭→メロ制約・A近傍）は依存薄・歌モノ実需直撃で P1〜P3 の裏で小実験可。

**順序の芯**：燃料装置の最小実証(P2は薄く先行可) → 品質floorの器(P1) → 苦手直撃の操作面(P3) → 全体足場(P4)。ただし **P0 は無条件で先**（低リスク・即メロが書きやすくなる）。

## 5. リスク/罠（哲学を裏切る落とし穴・Bより）
- ループのtop-kを1本に潰して自動採用＝**"候補まで・仕上げは人間"を裏切る**。常にspread＋信頼度。**総合スコア非表示**（E-rule total ⊥ 耳）。
- E-rule総合点の最適化＝gaming（`self-check-log` R2 実証）。項目別critiqueで直す・個人E-corpusで並べるのみ。
- 個人コーパスを**リテラル採譜で汚す**な（MIRメロ採譜は弱い）。メロは統計/傾向＋信頼度ゲート。堅いコアループ(コード)を主軸に。
- 著作権：自作=リテラル可、参考曲mp3は**統計/派生事実のみ・音源は解析後削除**（著30-4）。project/library分離厳守。
- 外部モデルサーバー建設の誘惑（FMD/Audiobox等）＝**贅沢品・後回し**。純TS二本立てで足りる。
- 表面のmicro最適化沼（耳なしchurnは退行・`self-check-log`）＝T2は「層を人に開く」方向で、機械が表面を"正解に寄せる"方向に振らない。

## 6. やらないこと
完成トラック一括生成(Suno的)／候補を総合スコアで1本自動採用／外部ニューラルメロAI導入／常設外部評価サーバー／ドラム・ベース生成強化(メロより後)／厳密対位法拘束／アーティスト完全物まね／他者コーパスのリテラル保存。

## 7. 未決＝オーナーの選択（着手点）
P0（足場の即効ウィン）は無条件で価値が高く低リスク＝**まず P0 から**が推奨。その上で**次にどのアークへ賭けるか**＝(i) P1 自己進化ループ常設（品質floor・部品済）か、(ii) P2 自作コーパス（燃料・戦略的）か、(iii) P3 メロ層操作（苦手直撃の手触り）か、は**作曲家＝オーナーの制作実感で決めるべき**。→ 会話で確認して本計画のPhase順を確定し、Task化する。
